# Modbus 类对象优化和修复方案

**目标文件**: `src/Modbus/_bak/cModbusMaster.cls` 和 `src/Modbus/_bak/cModbusSlave.cls`

**相关依赖**:
- `src/Winsock/cWinSock.cls` - Socket通信
- `src/Websocket/cByteBuffer.cls` - 字节缓冲区
- `src/Tools/DateTime/Timer/cTimer.cls` - 定时器（用于替代阻塞循环）

## 一、cModbusMaster.cls 问题与修复

### 1.1 RTU响应长度计算错误 [严重]

**位置**: 第1104-1116行 `ParseRTUFrame` 函数

**问题**: 写单线圈/寄存器的响应长度设置为6，应该是8（含CRC）

**修复**:
```vb
Case MB_MASTER_FC_WRITE_SINGLE_COIL, MB_MASTER_FC_WRITE_SINGLE_REGISTER
    ' 写单个响应: SlaveID(1) + FC(1) + Addr(2) + Value(2) + CRC(2) = 8
    lPDULen = 8  ' 原来是6

Case MB_MASTER_FC_WRITE_MULTIPLE_COILS, MB_MASTER_FC_WRITE_MULTIPLE_REGISTERS
    ' 写多个响应: SlaveID(1) + FC(1) + Addr(2) + Quantity(2) + CRC(2) = 8
    lPDULen = 8  ' 原来是6
```

---

### 1.2 TCP响应解析数组越界 [严重]

**位置**: 第1163行 `SendRequestTCP` 函数

**问题**: `ReDim baResponse(lMBAPLen)` 多分配了一个字节

**修复**:
```vb
' 修复前
ReDim baResponse(lMBAPLen) As Byte

' 修复后
ReDim baResponse(lMBAPLen - 1) As Byte
```

---

### 1.3 TCP BuildFrame 中 Unit ID 重复 [严重]

**位置**: 第966-984行 `BuildFrame` 函数

**问题**: MBAP头中设置了UnitID，但复制PDU时又包含了SlaveID，导致数据错位

**修复**:
```vb
' TCP: MBAP Header (7 bytes) + PDU (不含SlaveID)
m_TransactionID = m_TransactionID + 1
If m_TransactionID > 65535 Then m_TransactionID = 1

' 修复: 帧大小 = MBAP(7) + PDU长度 - 1 (因为UnitID在MBAP中)
ReDim baFrame(m_Defaults.TCP_MBAP_SIZE + lPDULen - 2) As Byte

' MBAP Header
baFrame(0) = (m_TransactionID \ 256) And &HFF
baFrame(1) = m_TransactionID And &HFF
baFrame(2) = 0
baFrame(3) = 0
baFrame(4) = ((lPDULen) \ 256) And &HFF
baFrame(5) = (lPDULen) And &HFF
baFrame(6) = PDU(0)  ' Unit ID

' PDU (跳过SlaveID，只复制FC和数据)
If lPDULen > 1 Then
    CopyMemory baFrame(7), PDU(1), lPDULen - 1
End If
```

### 1.4 写响应验证不完整 [严重]

**位置**: 第604-608行 `WriteSingleCoil` 函数（WriteSingleRegister同样问题）

**问题**: 只验证了5个字节，应验证完整的6字节

**修复**:
```vb
If UBound(baResponse) >= 5 Then
    If baResponse(0) = baRequest(0) And _
       baResponse(1) = baRequest(1) And _
       baResponse(2) = baRequest(2) And _
       baResponse(3) = baRequest(3) And _
       baResponse(4) = baRequest(4) And _
       baResponse(5) = baRequest(5) Then   ' 添加第6个字节验证
        WriteSingleCoil = True
    End If
End If
```

### 1.6 GetTickCount 溢出问题 [中等]

**位置**: 第1071行

**修复**: 添加辅助函数处理溢出
```vb
Private Function GetElapsedTime(ByVal lStartTime As Long) As Long
    Dim lNow As Long
    lNow = GetTickCount
    If lNow >= lStartTime Then
        GetElapsedTime = lNow - lStartTime
    Else
        GetElapsedTime = (2147483647 - lStartTime) + lNow + 1
    End If
End Function
```

## 二、cModbusSlave.cls 问题与修复

### 2.1 WriteMultipleCoils 请求偏移错误 [严重]

**位置**: 第1380行 `ProcessWriteMultipleCoils` 函数

**问题**: ByteCount位置计算错误，`lPDUOffset + 5` 应为 `lPDUOffset + 4`

**修复**:
```vb
' FC15请求格式: SlaveID(0) + FC(1) + AddrHi(2) + AddrLo(3) + QtyHi(4) + QtyLo(5) + ByteCount(6) + Data(7...)
' 相对于PDUOffset: Addr(0,1) + Qty(2,3) + ByteCount(4) + Data(5...)
lByteCount = Request(lPDUOffset + 4)  ' 原来是 + 5

' 数据起始位置也要调整
For i = 0 To lQuantity - 1
    bByte = Request(lPDUOffset + 5 + i \ 8)  ' 原来是 + 6
    ' ...
Next i
```

---

### 2.2 RTU帧长度计算函数过于简单 [严重]

**位置**: 第1543-1547行 `CalculateFrameLength` 函数

**问题**: 固定返回8，不支持变长帧

**修复**:
```vb
Private Function CalculateFrameLength(ByVal FC As Byte) As Long
    Select Case FC
        Case MB_SLAVE_FC_READ_COILS, MB_SLAVE_FC_READ_DISCRETE_INPUTS, _
             MB_SLAVE_FC_READ_HOLDING_REGISTERS, MB_SLAVE_FC_READ_INPUT_REGISTERS
            CalculateFrameLength = 8

        Case MB_SLAVE_FC_WRITE_SINGLE_COIL, MB_SLAVE_FC_WRITE_SINGLE_REGISTER
            CalculateFrameLength = 8

        Case MB_SLAVE_FC_WRITE_MULTIPLE_COILS, MB_SLAVE_FC_WRITE_MULTIPLE_REGISTERS
            If m_RTUBuffer.Size >= 7 Then
                Dim lByteCount As Long
                lByteCount = m_RTUBuffer.PeekByte(6)
                CalculateFrameLength = 7 + lByteCount + 2
            Else
                CalculateFrameLength = 0  ' 等待更多数据
            End If

        Case Else
            CalculateFrameLength = 8
    End Select
End Function
```

### 2.3 RTU异常响应数组大小错误 [严重]

**位置**: 第1520行 `BuildExceptionResponse` 函数

**问题**: `ReDim baResponse(2 + 1)` 分配4字节，但RTU异常响应需要5字节

**修复**:
```vb
' RTU异常响应 = SlaveID(1) + FC|0x80(1) + ExCode(1) + CRC(2) = 5字节
ReDim baResponse(4) As Byte  ' 索引0-4
baResponse(0) = m_SlaveID
bFC = Request(1)
baResponse(1) = bFC Or &H80
baResponse(2) = ExceptionCode

Dim wCRC As Long
wCRC = ModbusCRC16(baResponse, 0, 3)  ' 计算前3个字节
baResponse(3) = wCRC And &HFF
baResponse(4) = (wCRC \ 256) And &HFF
```

### 2.6 RTU阻塞循环 [严重]

**位置**: 第716-734行 `StartRTUProcessing` 函数

**问题**: `Do...Loop` 阻塞主线程，VB6无法处理其他事件

**修复**: 使用 `cTimer` 替代阻塞循环

```vb
' 添加成员变量
Private WithEvents m_RTUTimer As cTimer

' 修改 StartRTUProcessing
Private Sub StartRTUProcessing()
    Set m_RTUTimer = New cTimer
    m_RTUTimer.Interval = 10  ' 10ms轮询
    m_RTUTimer.Enabled = True
End Sub

' 添加Timer事件
Private Sub m_RTUTimer_Timer()
    Dim baBuffer(255) As Byte
    Dim lBytesRead As Long

    On Error GoTo ErrorHandler

    If Not m_RTUProcessing Or m_State <> MB_SLAVE_STATE_RUNNING Then
        m_RTUTimer.Enabled = False
        Exit Sub
    End If

    ReadFile m_hSerialPort, baBuffer(0), 256, lBytesRead, 0

    If lBytesRead > 0 Then
        Dim baTemp() As Byte
        ReDim baTemp(lBytesRead - 1) As Byte
        CopyMemory baTemp(0), baBuffer(0), lBytesRead
        m_RTUBuffer.Append baTemp

        Do
            Dim baFrame() As Byte
            baFrame = ParseAndProcessRTUFrame()
            If UBound(baFrame) < 0 Then Exit Do
        Loop
    End If
    Exit Sub
ErrorHandler:
    RaiseError "RTUProcessing error: " & Err.Description
End Sub

' 修改 StopRTU
Private Sub StopRTU()
    On Error Resume Next
    m_RTUProcessing = False
    If Not m_RTUTimer Is Nothing Then
        m_RTUTimer.Enabled = False
        Set m_RTUTimer = Nothing
    End If
    ' ... 其余清理代码
End Sub
```

## 四、建议的额外改进

1. **日志接口**: 添加 `Event OnLog(Level, Message)` 替代 `Debug.Print`
2. **重连机制**: cModbusMaster 添加自动重连功能，使用 cTimer
3. **LastError属性**: 添加 `LastErrorCode` 和 `LastErrorDescription` 属性供调用者查询
