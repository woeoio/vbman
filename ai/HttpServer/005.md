# HttpServer Chunked 编码独立解析器设计方案

## 背景

基于 003.md 和 004.md 的分析，Chunked 传输编码的实现复杂度是 Content-Length 的 3-4 倍。为了降低代码耦合度和维护成本，建议将 Chunked 解析逻辑独立实现为一个专用类。

## 设计目标

1. **高内聚低耦合**：Chunked 解析逻辑封装在独立类中，不污染主请求处理流程
2. **可测试性**：可以独立单元测试，无需启动完整 HttpServer
3. **可维护性**：状态机清晰，边界条件易于追踪
4. **性能**：零拷贝设计，最小化内存分配

## 核心设计

### 类结构

```vb
' cHttpChunkedDecoder.cls
Option Explicit

' 解析状态枚举
Private Enum ParseState
    State_Header      ' 解析请求头
    State_ChunkSize   ' 解析 chunk-size
    State_ChunkCRLF   ' 等待 chunk-size 后的 CRLF
    State_ChunkData   ' 读取 chunk-data
    State_ChunkEndCRLF ' 等待 chunk-data 后的 CRLF
    State_Trailer     ' 解析 trailer headers
    State_Complete    ' 解析完成
End Enum

' 缓冲区
Private buffer() As Byte          ' 累积接收的数据
Private bufferLen As Long         ' 缓冲区有效长度

' 解析状态
Private state As ParseState
Private chunkSize As Long         ' 当前 chunk 的大小
Private chunkSizeReceived As Long ' 当前 chunk 已接收的数据量
Private bodyData As New Collection ' 累积的 body 数据片段

' 解析结果
Private headers As Scripting.Dictionary ' 请求头
Private body As Byte()             ' 完整的请求体
Private trailer As Scripting.Dictionary ' trailer headers（可选）

' 错误信息
Private pvLastError As String

' 主要方法
Public Function Feed(data() As Byte) As Boolean
Public Function IsComplete() As Boolean
Public Function GetHeaders() As Scripting.Dictionary
Public Function GetBody() As Byte()
Public Function GetTrailer() As Scripting.Dictionary
Public Property Get LastError() As String

' 内部解析方法
Private Function ParseHeader() As Boolean
Private Function ParseChunkSize() As Boolean
Private Function ParseChunkData() As Boolean
Private Function ParseTrailer() As Boolean
```

### 使用方式

```vb
' 在 cHttpServer.cls 的 OnDataArrival 中

' 1. 判断是否需要 Chunked 解析
If Request.Header.Exists("Transfer-Encoding") And _
   InStr(LCase(Request.Header("Transfer-Encoding")), "chunked") > 0 Then
    
    ' 2. 使用 Chunked 解析器
    Dim chunkedDecoder As New cHttpChunkedDecoder
    
    ' 3. 持续喂数据
    Do While HasMoreData
        data = GetAvailableData()
        If chunkedDecoder.Feed(data) = False Then
            ' 解析错误
            response.State400 chunkedDecoder.LastError
            Exit Sub
        End If
        
        ' 4. 检查是否完成
        If chunkedDecoder.IsComplete() Then
            ' 5. 获取完整请求
            Set Request.Header = chunkedDecoder.GetHeaders()
            Request.RawBodyBin = chunkedDecoder.GetBody()
            Exit Do
        End If
    Loop
Else
    ' 使用 Content-Length 方式
    ' ... 原有逻辑
End If
```

## 状态机详解

### 状态流转图

```
State_Header
    │
    ├─ 找到 CRLFCRLF → 解析头部 → State_ChunkSize
    └─ 未找到       → 等待更多数据

State_ChunkSize
    │
    ├─ 找到 CRLF   → 解析十六进制大小 → State_ChunkCRLF
    └─ 未找到      → 等待更多数据

State_ChunkCRLF
    │
    ├─ 收到 CRLF   → State_ChunkData
    └─ 数据不足    → 等待更多数据

State_ChunkData
    │
    ├─ 已接收 chunkSize 字节 → State_ChunkEndCRLF
    └─ 未收完                → 继续接收

State_ChunkEndCRLF
    │
    ├─ 收到 CRLF
    │   ├─ chunkSize = 0 → State_Trailer
    │   └─ chunkSize > 0 → State_ChunkSize
    └─ 数据不足          → 等待更多数据

State_Trailer
    │
    ├─ 找到 CRLFCRLF → 解析 trailer → State_Complete
    └─ 未找到        → 等待更多数据

State_Complete
    │
    └─ 解析完成，可以获取结果
```

### 关键状态说明

#### State_ChunkSize
- **输入**：`"1a3\r\n"` (十六进制 + CRLF)
- **处理**：提取十六进制数字，转换为十进制
- **输出**：`chunkSize = 419` (0x1a3)

#### State_ChunkData
- **边界**：chunk-data 可能跨越多个 TCP 包
- **优化**：使用零拷贝，只记录数据位置，不立即拼接

#### State_Trailer
- **可选**：很多 chunked 响应没有 trailer
- **处理**：与请求头相同的 key:value 格式

## 错误处理设计

### 错误类型

```vb
' 无效格式
"Invalid chunk format: missing CRLF"
"Invalid chunk size: not hexadecimal"
"Invalid chunk size: too large"

' 数据不完整
"Unexpected end of data in chunk header"
"Chunk data truncated"

' 协议违规
"Chunk size mismatch: declared {declared}, received {received}"
"Invalid trailer format"

' 内存限制
"Chunk size exceeds maximum allowed: {size}"
```

### 错误恢复策略

1. **解析失败**：返回 False，设置 LastError
2. **数据不完整**：保持当前状态，等待更多数据
3. **协议违规**：终止解析，清空缓冲区
4. **内存超限**：返回错误，防止 DoS 攻击

## 性能优化

### 1. 零拷贝设计

```vb
' 不立即拼接 body 数据
' 而是记录每个 chunk 的引用
Private Type ChunkRef
    data() As Byte      ' 原始数据
    offset As Long      ' 在 data 中的偏移
    length As Long      ' 本 chunk 的长度
End Type

Private chunks As New Collection  ' 存储 ChunkRef

' 最后需要时再拼接
Private Function BuildBody() As Byte()
    Dim totalLen As Long
    Dim result() As Byte
    Dim pos As Long
    
    ' 计算总长度
    For Each ref In chunks
        totalLen = totalLen + ref.length
    Next
    
    ' 一次性分配内存
    ReDim result(0 To totalLen - 1)
    
    ' 复制数据
    For Each ref In chunks
        CopyMemory result(pos), ref.data(ref.offset), ref.length
        pos = pos + ref.length
    Next
    
    BuildBody = result
End Function
```

### 2. 缓冲区管理

```vb
' 动态扩展缓冲区
Private Sub EnsureBufferCapacity(minSize As Long)
    If UBound(buffer) < minSize Then
        ReDim Preserve buffer(0 To minSize * 1.5)  ' 1.5倍增长
    End If
End Sub

' 消耗已处理的数据
Private Sub ConsumeProcessedBytes(count As Long)
    If count >= bufferLen Then
        bufferLen = 0
    Else
        ' 移动剩余数据到缓冲区开头
        CopyMemory buffer(0), buffer(count), bufferLen - count
        bufferLen = bufferLen - count
    End If
End Sub
```

## 集成到 HttpServer 的步骤

### 步骤1：创建解析器类

创建 `src/HttpServer/cHttpChunkedDecoder.cls`

### 步骤2：修改 cClientCallback

```vb
' 在 cClientCallback 中添加
Public chunkedDecoder As cHttpChunkedDecoder
```

### 步骤3：修改 OnDataArrival

```vb
' 在解析头部后
If Request.Header.Exists("Transfer-Encoding") And _
   InStr(LCase(Request.Header("Transfer-Encoding")), "chunked") > 0 Then
    
    If oCallback.chunkedDecoder Is Nothing Then
        Set oCallback.chunkedDecoder = New cHttpChunkedDecoder
    End If
    
    ' 喂数据
    If oCallback.chunkedDecoder.Feed(aReqBody) = False Then
        response.State400 oCallback.chunkedDecoder.LastError
        Exit Sub
    End If
    
    ' 检查是否完成
    If Not oCallback.chunkedDecoder.IsComplete() Then
        ' 等待更多数据
        Exit Sub
    End If
    
    ' 获取完整 body
    aReqBody = oCallback.chunkedDecoder.GetBody()
    
    ' 清理
    Set oCallback.chunkedDecoder = Nothing
End If

' 继续原有逻辑
If Request.ParseBody(aReqBody) = False Then
    ERR.Raise 500, "REQUEST", Request.LastError
End If
```

### 步骤4：清理资源

```vb
' 在 OnCloseSck 中
Set oCallback.chunkedDecoder = Nothing
```

## 注意事项

1. **内存限制**：设置最大 body 大小限制，防止 DoS
   ```vb
   Const MAX_BODY_SIZE = 10 * 1024 * 1024  ' 10MB
   ```

2. **超时处理**：chunked 请求应该有超时机制
   ```vb
   ' 记录最后接收时间
   oCallback.lastRecvTime = Now
   ' 定时检查超时
   ```

3. **错误恢复**：解析失败后应该清空缓冲区
   ```vb
   If parseFailed Then
       bufferLen = 0
       state = State_Header
   End If
   ```

4. **性能监控**：记录解析耗时，发现性能瓶颈
   ```vb
   Debug.Print "Chunked parse time: " & (Timer - startTime) * 1000 & "ms"
   ```
