## 一、循环引用导致内存泄漏（严重）

VB6 使用引用计数进行垃圾回收，循环引用会导致对象的引用计数永远不为 0，从而永远无法销毁。这是本审查发现的最关键问题。

### 循环引用链 1：`cClientCallback` ↔ `cHttpServer`

```
cClientCallback.Parent ──→ cHttpServer
cHttpServer.m_cConnPool ──→ Collection ──→ cClientCallback
```

**出处**：

- `cClientCallback.cls:23` — `Public Parent As cHttpServer`
- `cHttpServer.cls:132` — `Set .Parent = Me`

`cClientCallback` 通过 `Parent` 持有 `cHttpServer`，而 `cHttpServer` 通过 `m_cConnPool` 集合持有 `cClientCallback`，形成双向引用环。

### 循环引用链 2：`cClientCallback` → `cHttpServerContext` → `cHttpServerResponse` → `cClientCallback`

```
cClientCallback.Context      ──→ cHttpServerContext
cHttpServerContext.Response   ──→ cHttpServerResponse
cHttpServerResponse.Client    ──→ cClientCallback    (通过 fClient)
cHttpServerResponse.Sck       ──→ cTlsReMaster       (额外持有 Socket)
```

**出处**：

- `cClientCallback.cls:22` — `Public Context As cHttpServerContext`
- `cHttpServer.cls:368` — `Set response.fClient = oCallback`
- `cHttpServer.cls:442` — `Set .Response = response`
- `cHttpServerResponse.cls:43-47` — `Friend Property Set fClient` 同时设置 `Client` 和 `Sck`

这是最严重的三角循环引用：`Client → Context → Response → Client`，且 Response 还额外持有 Socket 引用。

### 循环引用链 3：`cHttpServerContext` → `cSSEContext` → `cSSE` → `cClientCallback` → `cHttpServerContext`

```
cHttpServerContext.SSE     ──→ cSSEContext
cSSEContext.SSEInst        ──→ cSSE
cSSE.Clients(hSocket)      ──→ cClientCallback
cClientCallback.Context    ──→ cHttpServerContext
```

**出处**：

- `cHttpServerContext.cls:31` — `Public SSE As New cSSEContext`
- `cSSEContext.cls:18` — `Friend Property Set flsSSEInst(o As cSSE)`
- `cSSE.cls:130` — `Clients.Add Client.hSocket, Client`

形成 SSE 相关的四对象循环引用，SSE 客户端连接期间永远不会被释放。

### 循环引用链 4：`cHttpServerContext` 内部对象互相持有

```
cHttpServerContext 持有: Request, Response, Session, Cookies, Db, SSE, ClientInfo, UserData
cHttpServerResponse 通过 Client 可访问 Context 上的所有对象
cSSEContext 通过 SSEInst 可访问 cSSE，进而访问其他 Client 的 Context
```

整个对象图高度耦合，一个客户端的对象无法释放，会导致连带效应。

## 二、`OnCloseSck` 连接清理不彻底（严重）

代码中已标注了 `todo`（`cHttpServer.cls:492`）。

```vb
Friend Sub OnCloseSck(Index As Long)
    'todo 这里可能有内存泄漏
    Dim oCallback As cClientCallback
    Set oCallback = m_cConnPool.Item(Index)
    If oCallback.IsSSE = True Then
        SSE.CloseClient oCallback.hSocket
    End If
    
    ' 清理接收缓冲区
    oCallback.GetRecvBufferLen = 0
    
    Set oCallback = Nothing           ' ← 只断开局部变量引用
    '    Set m_cConnPool.Item(Index) = Nothing
    m_cConnPool.Remove Index         ' ← 从集合移除，但对象内部循环引用未断开
    If m_cConnPool.Count >= Index Then
        m_cConnPool.Add Nothing, Before:=Index
    End If
    Do While m_cConnPool.Count > 0
        If Not m_cConnPool.Item(m_cConnPool.Count) Is Nothing Then
            Exit Do
        End If
        m_cConnPool.Remove m_cConnPool.Count
    Loop
End Sub
```

### 问题清单

| # | 问题 | 说明 |
|---|------|------|
| 1 | `Set oCallback = Nothing` 无效 | 只断开局部变量引用，不影响对象本身的引用计数。`m_cConnPool` 中的对象在 `Remove` 前仍被集合引用 |
| 2 | 未清理 `oCallback.Context` 内部子对象 | Request, Response, Session, Cookies, Db, SSE 等子对象之间存在循环引用，必须显式释放 |
| 3 | 未清理 `oCallback.Socket` | Socket 的事件仍可能触发（`DataArrival`、`CloseSck`），可能导致访问已失效的对象 |
| 4 | SSE 双重清理风险 | `OnCloseSck` 调用 `SSE.CloseClient`，而 `SSE.CloseClient` 内部也会关闭 Socket 并清理，可能导致重复操作或遗漏 |
| 5 | 占位符逻辑问题 | `m_cConnPool.Add Nothing, Before:=Index` 仅在 `m_cConnPool.Count >= Index` 时执行，当 Count < Index 时不会填充占位，可能导致 Index 错位 |

## 三、`StopMe()` 方法问题

```vb
Public Function StopMe() As Boolean
    On Error GoTo EH
    m_oServer.CloseSck
    IsRunning = False
    'clear
    Set m_cConnPool = Nothing       ' ← 直接置空集合，未逐一清理内部对象
    Set HttpApp = Nothing
    Set HttpSvr = Nothing
    Set m_oServer = Nothing
    
    Set SSE = Nothing               ' ← SSE 客户端未逐一关闭
    
    Set Router = Nothing
    Set RouteBefore = Nothing
    Set RouteAfter = Nothing
    Set Database = Nothing
    
    Set CrossDomain = Nothing
    
    StopMe = True
    Exit Function
EH:
    LastError = ERR.Number & "#" & ERR.Description
End Function
```

| # | 问题 | 说明 |
|---|------|------|
| 1 | `m_cConnPool` 直接置空 | 池中每个 `cClientCallback` 及其子对象（Socket, Context, Response 等）都没有显式释放，由于循环引用无法自动销毁 |
| 2 | SSE 客户端未逐一关闭 | 应先遍历 `SSE.Clients` 关闭所有 SSE 连接，否则 SSE 持有的 `cClientCallback` 引用不会释放 |
| 3 | `ScriptEngine` 未清理 | 声明为 `Public ScriptEngine As New cScriptEngine`，`StopMe` 中没有 `Set ScriptEngine = Nothing` |
| 4 | `As New` 声明隐患 | `m_cConnPool As New Collection` 被置 Nothing 后，VB6 的 `As New` 特性会在下次访问时自动重新创建。虽然 `IsRunning = False` 做了逻辑保护，但建议改为显式创建 |

## 四、`ProcessHttpRequest` 中局部对象未显式释放

```vb
Private Sub ProcessHttpRequest(Index As Long, completeRequest As Variant)
    Dim Request As New cHttpServerRequest
    Dim response As New cHttpServerResponse
    Dim Session As New cHttpServerSession
    Dim Cookies As New cHttpServerCookies
    
    ' ... 赋值到 oCallback.Context ...
    With oCallback.Context
        Set .Request = Request
        Set .response = response
        Set .Session = Session
        Set .Cookies = Cookies
        Set .Db = oCallback.Parent.Database
        Set .SSE.flsSSEInst = SSE
    End With
    
    ' ... 路由处理 ...
    
    ' 方法结束，局部变量自动释放
    ' 但 Request/Response/Session/Cookies 已被 Context 持有，生命周期随 Context 延续
    ' 由于循环引用，Context 永远不会被销毁，这些对象也永远不会被释放
End Sub
```

每个 HTTP 请求创建的对象（Request, Response, Session, Cookies）被 Set 到 Context 上，请求结束后不会随局部变量释放，而是随 Context 续命。而 Context 因循环引用永不销毁，所以这些对象也永远不会释放。

## 五、`cHttpServerResponse` 冗余持有 Socket 引用

```vb
' cHttpServerResponse.cls
Dim Client As cClientCallback
Dim Sck As cTlsReMaster          ' ← 冗余引用
Dim CrossDomain As cHttpCrossDomain  ' ← 冗余引用

Friend Property Set fClient(o As Object)
    Set Client = o
    Set Sck = Client.Socket      ' ← 额外持有 Socket
    Set CrossDomain = Client.Parent.CrossDomain  ' ← 额外持有 CrossDomain
End Property
```

Response 已经持有 `Client` 引用，完全可以通过 `Client.Socket` 和 `Client.Parent.CrossDomain` 访问 Socket 和 CrossDomain，不需要单独持有。额外持有引用：

1. 增加了引用计数，使对象更难释放
2. 加剧了循环引用链的复杂度
3. 如果 Socket 或 CrossDomain 被替换，Response 持有的是旧引用

## 六、`cSSE.Data` 共享状态并发安全问题

```vb
' cSSE.cls
Public Data As New Scripting.Dictionary    ' ← 公共共享可变状态

Public Function SendPack(Action As String, Content As Variant, Optional hSocket As Long) As Boolean
    Data.RemoveAll              ' ← 先清空
    Data.Add "action", Action   ' ← 再填充
    Data.Add "content", Content
    SendPack = Send(hSocket)    ' ← 再发送
End Function

Public Function Send(Optional ByVal hSocket As Long) As Boolean
    ' ...
    SseDataText = ... & Json.Encode(Data) & ...   ' ← 使用 Data
    ' ...
QH:
    hSocketByUser = 0
    hSocketsByGroup.RemoveAll
    Data.RemoveAll              ' ← 发送完清空
End Function
```

`Data` 是 `cSSE` 的公共成员，`SendPack` 先 `RemoveAll` 再 `Add`，在多客户端并发场景下：

1. 请求 A 正在 `Send` 中使用 `Data` 编码 JSON
2. 请求 B 调用 `SendPack` 将 `Data` 清空重新填充
3. 请求 A 编码出错误数据或触发运行时错误

此外，`cSSE.Entry` 中 `Data.Add "hello", "init"` 也会累积到共享 Data 中，如果后续 `Send` 前未清空，可能包含意外数据。

## 九、修复方案

### 核心思路

**在对象生命周期结束时，手动断开循环引用。** VB6 无法自动处理循环引用，必须在业务层面主动管理对象释放顺序。

### 9.1 为 `cClientCallback` 添加 `Release` 方法

```vb
' cClientCallback.cls — 新增
Friend Sub Release()
    ' 先断开 Context 内部循环引用
    If Not Context Is Nothing Then
        If Not Context.Request Is Nothing Then Set Context.Request = Nothing
        If Not Context.Response Is Nothing Then
            ' 断开 Response → Client 的引用
            Set Context.Response.fClient = Nothing  ' 需要增加一个 Friend 属性
            Set Context.Response = Nothing
        End If
        If Not Context.Session Is Nothing Then Set Context.Session = Nothing
        If Not Context.Cookies Is Nothing Then Set Context.Cookies = Nothing
        If Not Context.Db Is Nothing Then Set Context.Db = Nothing
        If Not Context.SSE Is Nothing Then Set Context.SSE = Nothing
        If Not Context.ClientInfo Is Nothing Then Set Context.ClientInfo = Nothing
        If Not Context.Server Is Nothing Then Set Context.Server = Nothing
        If Not Context.App Is Nothing Then Set Context.App = Nothing
        If Not Context.UserData Is Nothing Then
            Context.UserData.RemoveAll
            Set Context.UserData = Nothing
        End If
        Set Context = Nothing
    End If
    ' 断开 Parent 引用（解开 cClientCallback ↔ cHttpServer 循环）
    Set Parent = Nothing
    ' 关闭并释放 Socket
    If Not Socket Is Nothing Then
        On Error Resume Next
        Socket.CloseSck
        On Error GoTo 0
        Set Socket = Nothing
    End If
    ' 清空缓冲区
    recvBufferLen = 0
End Sub
```

### 9.2 为 `cHttpServerResponse` 添加释放属性

```vb
' cHttpServerResponse.cls — 新增
Friend Property Set fClient(o As Object)
    If o Is Nothing Then
        Set Client = Nothing
        Set Sck = Nothing
        Set CrossDomain = Nothing
    Else
        Set Client = o
        Set Sck = Client.Socket
        Set CrossDomain = Client.Parent.CrossDomain
    End If
End Property
```

### 9.3 修改 `OnCloseSck`

```vb
Friend Sub OnCloseSck(Index As Long)
    Dim oCallback As cClientCallback
    Set oCallback = m_cConnPool.Item(Index)
    
    If oCallback.IsSSE = True Then
        SSE.CloseClient oCallback.hSocket
    End If
    
    ' 主动断开所有循环引用
    oCallback.Release
    
    m_cConnPool.Remove Index
    
    ' 清理尾部空位
    Do While m_cConnPool.Count > 0
        If Not m_cConnPool.Item(m_cConnPool.Count) Is Nothing Then
            Exit Do
        End If
        m_cConnPool.Remove m_cConnPool.Count
    Loop
End Sub
```

### 9.4 修改 `StopMe()`

```vb
Public Function StopMe() As Boolean
    On Error GoTo EH
    m_oServer.CloseSck
    IsRunning = False
    
    ' 先逐一释放所有连接（断开循环引用）
    Dim i As Long
    For i = m_cConnPool.Count To 1 Step -1
        Dim oCallback As cClientCallback
        Set oCallback = m_cConnPool.Item(i)
        If Not oCallback Is Nothing Then
            oCallback.Release
        End If
    Next
    Set m_cConnPool = Nothing
    
    Set HttpApp = Nothing
    Set HttpSvr = Nothing
    Set m_oServer = Nothing
    
    ' 清理 SSE（此时 Clients 已在 Release 中处理）
    Set SSE = Nothing
    Set ScriptEngine = Nothing
    
    Set Router = Nothing
    Set RouteBefore = Nothing
    Set RouteAfter = Nothing
    Set Database = Nothing
    
    Set CrossDomain = Nothing
    
    StopMe = True
    Exit Function
EH:
    LastError = ERR.Number & "#" & ERR.Description
End Function
```

## 十、循环引用关系图

```
┌─────────────┐     m_cConnPool      ┌──────────────────┐
│  cHttpServer │◄────────────────────│  cClientCallback  │
│              │─────────────────────►│                   │
└─────────────┘     Parent           │  Context ─────────┼───┐
                                      │  Socket           │   │
                                      │  Parent ──────────┼───┘ (回到 cHttpServer)
                                      └────────┬──────────┘
                                               │
                                               ▼
                                      ┌──────────────────┐
                                      │ cHttpServerContext│
                                      │                   │
                                      │  Request          │
                                      │  Response ────────┼───┐
                                      │  Session          │   │
                                      │  Cookies          │   │
                                      │  SSE ─────┐       │   │
                                      │  Db       │       │   │
                                      └───────────┼───────┘   │
                                                  │           │
                    ┌─────────────────────────────┘           │
                    ▼                                         ▼
           ┌─────────────────┐                     ┌──────────────────┐
           │  cSSEContext     │                     │ cHttpServerResponse│
           │                  │                     │                    │
           │  SSEInst ────────┼──┐                  │  Client ──────────┼───► cClientCallback
           └──────────────────┘  │                  │  Sck (冗余)       │     (回到上方)
                                 ▼                  └──────────────────┘
                          ┌─────────────────┐
                          │      cSSE        │
                          │                  │
                          │  Clients ────────┼──► cClientCallback (又回到顶部)
                          └─────────────────┘
```

**关键断开点**：在 `cClientCallback.Release()` 中，必须按以下顺序断开：
1. `Context.Response.fClient = Nothing` — 断开 Response → Client 引用
2. `Context.*` 全部置 Nothing — 断开 Context 内部引用
3. `Context = Nothing` — 断开 Client → Context 引用
4. `Parent = Nothing` — 断开 Client → cHttpServer 引用
5. `Socket = Nothing` — 断开事件源

### 关键修改详解

#### 1. cClientCallback.Release() — 断开所有循环引用的核心方法

```vb
Friend Sub Release()
    ' 1. 断开 Response → Client 的循环引用
    If Not Context Is Nothing Then
        If Not Context.Response Is Nothing Then
            Set Context.Response.fClient = Nothing
        End If
        ' 2. 释放 Context 内部所有对象引用
        Context.Release
        Set Context = Nothing
    End If
    ' 3. 断开 Client → cHttpServer 的引用
    Set Parent = Nothing
    ' 4. 释放 ClientInfo
    Set ClientInfo = Nothing
    ' 5. 关闭并释放 Socket
    If Not Socket Is Nothing Then
        On Error Resume Next
        Socket.CloseSck
        On Error GoTo 0
        Set Socket = Nothing
    End If
    ' 6. 清空接收缓冲区
    recvBufferLen = 0
    IsSSE = False
End Sub
```

释放顺序至关重要：
- 必须先断开 `Response.fClient`（解开三角循环）
- 然后释放 Context 内部对象
- 再断开 Parent 引用（解开双向循环）
- 最后关闭 Socket（断开事件源）

#### 2. Socket 事件安全检查

```vb
Private Sub Socket_DataArrival(ByVal bytesTotal As Long)
    dataTotal = bytesTotal
    If Not Parent Is Nothing Then Parent.OnDataArrival Index
End Sub

Private Sub Socket_CloseSck()
    If Not Parent Is Nothing Then Parent.OnCloseSck Index
End Sub
```

防止 `Release()` 断开 Parent 后，Socket 事件仍触发导致访问已释放对象。

#### 3. OnCloseSck 调用顺序

```
SSE.CloseClient → 处理 OnClose 事件、清理 SSE 映射
    ↓
oCallback.Release → 断开循环引用、关闭 Socket
    ↓
m_cConnPool.Remove → 从集合移除
```
