# cWinsock TLS 支持设计

## 一、需求概述

为 cWinsock 添加 TLS/SSL 能力，使其 `Listen` 和 `Connect` 均可使用加密连接。

## 二、核心方案：内部替换 cAsyncSocket → cTlsSocket

### 2.1 可行性分析

| 对比项 | cAsyncSocket | cTlsSocket |
|--------|-------------|------------|
| 事件 | OnResolve/OnAccept/OnClose/OnConnect/OnReceive/OnSend/OnError/OnMessagePending/BeforeNotify/AfterNotify | **完全相同** + 多一个 OnCertificate |
| 方法 | Create/Listen/Accept/Connect/Close_/SendText/SendArray/ReceiveArray/Sync* | **完全兼容**（TLS 未启用时行为一致） |
| 来源 | 同一作者 wqweto，VbAsyncSocket 项目 | cAsyncSocket 的 TLS 扩展版 |

**结论：** cTlsSocket 是 cAsyncSocket 的严格超集，TLS 未启用时行为完全一致，可以直接替换。

### 2.2 替换方式

```vb
' 改造前
Private WithEvents m_oSocket As cAsyncSocket

' 改造后
Private WithEvents m_oSocket As cTlsSocket
```

所有 `New cAsyncSocket` 改为 `New cTlsSocket`，事件处理器签名不变。

## 三、TLS 模式设计

### 3.1 枚举

```vb
Private Enum EnumTlsMode
    TlsNone          ' 未启用 TLS（默认，纯 TCP）
    TlsModeClient    ' 客户端 TLS（Connect 时启用加密，无需证书）
    TlsModeCertFile  ' 证书文件（Listen 时需出示证书）
    TlsModeSubject   ' Windows 证书存储
    TlsModeMemory    ' 内存证书集合
End Enum
```

### 3.2 为什么多一个 TlsModeClient？

cWinsock 有两种使用模式：

| 模式 | 场景 | 需要证书？ | 链式函数 |
|------|------|-----------|---------|
| **客户端 Connect** | 连接远程 TLS 服务器 | 通常不需要 | `Tls()` |
| **服务端 Listen** | 监听并出示证书给客户端 | **必须** | `TlsCertFile` / `TlsCertSubject` / `TlsCertMemory` |

客户端连接 TLS 服务器（如 wss://、https://）时，通常不需要本地证书，只需启用加密即可。所以需要一个无参数的 `Tls()` 函数。

### 3.3 四个链式函数

```vb
' 客户端 TLS（Connect 时启用加密，无需证书）
' 用途：连接 wss:// 服务器、TLS TCP 服务器等
Public Function Tls() As cWinsock
    m_TlsMode = TlsModeClient
    Set Tls = Me
End Function

' 证书文件模式（最常用）
Public Function TlsCertFile( _
    ByVal CertFile As String, _
    Optional ByVal Password As String, _
    Optional ByVal AlpnProtocols As String = "http/1.1") As cWinsock

    m_TlsMode = TlsModeCertFile
    m_TlsCertFile = CertFile
    m_TlsPassword = Password
    m_TlsAlpnProtocols = AlpnProtocols
    Set TlsCertFile = Me
End Function

' Windows 证书存储模式
Public Function TlsCertSubject( _
    ByVal CertSubject As String, _
    Optional ByVal AlpnProtocols As String = "http/1.1") As cWinsock

    m_TlsMode = TlsModeSubject
    m_TlsCertSubject = CertSubject
    m_TlsAlpnProtocols = AlpnProtocols
    Set TlsCertSubject = Me
End Function

' 内存证书集合模式（高级）
Public Function TlsCertMemory( _
    ByVal Certificates As Collection, _
    ByVal PrivateKey As Collection, _
    Optional ByVal AlpnProtocols As String = "http/1.1") As cWinsock

    m_TlsMode = TlsModeMemory
    Set m_TlsCertificates = Certificates
    Set m_TlsPrivateKey = PrivateKey
    m_TlsAlpnProtocols = AlpnProtocols
    Set TlsCertMemory = Me
End Function
```

### 3.4 新增成员变量

```vb
Private m_TlsMode As EnumTlsMode
Private m_TlsCertFile As String
Private m_TlsPassword As String
Private m_TlsCertSubject As String
Private m_TlsCertificates As Collection
Private m_TlsPrivateKey As Collection
Private m_TlsAlpnProtocols As String
```

## 四、改造要点

### 4.1 Connect 方法改造

```vb
Public Sub Connect(Optional RemoteHost As String, Optional ByVal RemotePort As Long)
    ' ... 现有逻辑 ...
    
    Set m_oSocket = New cTlsSocket
    If Not m_oSocket.Create(SocketType:=eSocketType) Then
        pvRaiseError m_oSocket.LastError
        Exit Sub
    End If
    
    m_eState = sckResolvingHost
    
    ' 根据TLS模式决定连接参数
    If m_TlsMode <> TlsNone Then
        ' TLS 连接：UseTls=True
        If Not m_oSocket.Connect(m_sRemoteHost, m_lRemotePort, UseTls:=True) Then
            pvRaiseError m_oSocket.LastError
            Exit Sub
        End If
    Else
        ' 普通 TCP 连接
        If Not m_oSocket.Connect(m_sRemoteHost, m_lRemotePort) Then
            pvRaiseError m_oSocket.LastError
            Exit Sub
        End If
    End If
End Sub
```

### 4.2 Listen 方法改造

```vb
Public Sub Listen(Optional ByVal Port As Long, Optional ByVal LocalAddress As String = "")
    ' ... 现有逻辑 ...
    
    Set m_oSocket = New cTlsSocket
    If Not m_oSocket.Create(SocketPort:=m_lLocalPort, SocketType:=eSocketType, SocketAddress:=LocalAddress) Then
        pvRaiseError m_oSocket.LastError
        Exit Sub
    End If
    
    ' TLS 服务端：在 Listen 前初始化证书
    If m_TlsMode <> TlsNone And m_TlsMode <> TlsModeClient Then
        Select Case m_TlsMode
            Case TlsModeCertFile
                If Not m_oSocket.InitServerTls(CertFile:=m_TlsCertFile, Password:=m_TlsPassword, AlpnProtocols:=m_TlsAlpnProtocols) Then
                    pvRaiseError m_oSocket.LastError
                    Exit Sub
                End If
            Case TlsModeSubject
                If Not m_oSocket.InitServerTls(CertSubject:=m_TlsCertSubject, AlpnProtocols:=m_TlsAlpnProtocols) Then
                    pvRaiseError m_oSocket.LastError
                    Exit Sub
                End If
            Case TlsModeMemory
                If Not m_oSocket.InitServerTls(Certificates:=m_TlsCertificates, PrivateKey:=m_TlsPrivateKey, AlpnProtocols:=m_TlsAlpnProtocols) Then
                    pvRaiseError m_oSocket.LastError
                    Exit Sub
                End If
        End Select
    End If
    
    If m_eProtocol = wskTCPProtocol Then
        If Not m_oSocket.Listen() Then
            pvRaiseError m_oSocket.LastError
            Exit Sub
        End If
    End If
    
    ' ... 现有逻辑 ...
End Sub
```

### 4.3 AcceptFrom 方法改造

服务端 Accept 时，需要将 TLS 标志传递给新创建的客户端 Socket：

```vb
Friend Sub AcceptFrom(oListenSocket As cTlsSocket, oParent As cWinsock)
    Set m_oSocket = New cTlsSocket
    
    ' 根据父级是否启用 TLS 决定 Accept 参数
    Dim bUseTls As Boolean
    bUseTls = (oParent.m_TlsMode <> TlsNone)
    
    If Not oListenSocket.Accept(m_oSocket, UseTls:=bUseTls) Then
        pvRaiseError oListenSocket.LastError
        Exit Sub
    End If
    
    ' 继承父级的 TLS 模式（仅用于状态标识）
    If bUseTls Then m_TlsMode = TlsModeClient
    
    ' ... 其余现有逻辑不变 ...
End Sub
```

### 4.4 Close_ 清理

```vb
' 在 Close_() 现有清理逻辑末尾追加：
m_TlsMode = TlsNone
m_TlsCertFile = ""
m_TlsPassword = ""
m_TlsCertSubject = ""
Set m_TlsCertificates = Nothing
Set m_TlsPrivateKey = Nothing
m_TlsAlpnProtocols = ""
```

### 4.5 SyncConnect 改造

同步连接也需要支持 TLS：

```vb
If m_TlsMode <> TlsNone Then
    SyncConnect = m_oSocket.SyncConnect(m_sRemoteHost, m_lRemotePort, Timeout, UseTls:=True)
Else
    SyncConnect = m_oSocket.SyncConnect(m_sRemoteHost, m_lRemotePort, Timeout)
End If
```

### 4.6 新增事件：OnCertificate

cTlsSocket 多了一个 `OnCertificate` 事件，用于客户端验证服务器证书。cWinsock 需要透传此事件：

```vb
' 新增公共事件
Public Event ServerCertificateVerify(Client As cWinsock, Issuers As Object, ByRef Confirmed As Boolean)

' 事件处理器
Private Sub m_oSocket_OnCertificate(Issuers As Object, Confirmed As Boolean)
    ' 默认信任服务器证书（Confirmed=True），用户可通过事件覆盖
    Confirmed = True
    If m_bIsAcceptedClient And Not m_oParentServer Is Nothing Then
        ' 客户端模式下由使用者处理
    Else
        RaiseEvent ServerCertificateVerify(Me, Issuers, Confirmed)
    End If
End Sub
```

## 五、使用示例

### 5.1 客户端 TLS 连接

```vb
Dim ws As New cWinsock
ws.Tls.Connect "example.com", 443
```

### 5.2 服务端 TLS 监听

```vb
Dim svr As New cWinsock
svr.TlsCertFile("C:\certs\server.pfx", "password").Listen 443
```

### 5.3 条件启用

```vb
If Config.UseTLS Then
    ws.Tls
End If
ws.Connect Config.Host, Config.Port
```

### 5.4 链式配置

```vb
Dim svr As New cWinsock
svr.TlsCertFile("cert.pem").Delimiter(vbCrLf).AutoHeartbeat = True
svr.Listen 9000
```

## 六、改造清单

| 改动点 | 说明 |
|--------|------|
| `Private WithEvents m_oSocket As cAsyncSocket` → `cTlsSocket` | 核心替换 |
| 所有 `New cAsyncSocket` → `New cTlsSocket` | 4处（Connect、Listen、AcceptFrom、GetData 工具方法） |
| 新增枚举 `EnumTlsMode` + 7 个成员变量 | TLS 状态 |
| 新增 4 个链式函数 | Tls/TlsCertFile/TlsCertSubject/TlsCertMemory |
| 改造 `Connect` | TLS 模式时 UseTls:=True |
| 改造 `Listen` | TLS 模式时先 InitServerTls |
| 改造 `AcceptFrom` | 参数类型 cAsyncSocket→cTlsSocket，Accept 传 UseTls |
| 改造 `Close_` | 重置 TLS 状态 |
| 改造 `SyncConnect` | TLS 模式时 UseTls:=True |
| 新增事件 `ServerCertificateVerify` | 透传 cTlsSocket 的 OnCertificate |

## 七、注意事项

1. **InitServerTls 参数传递**：需验证 cTlsSocket.InitServerTls 的命名参数调用方式与 cAsyncSocket 方法签名兼容
2. **AcceptFrom 可见性**：`oParent.m_TlsMode` 是 Private 成员，AcceptFrom 需要改为 Friend 访问或新增 Friend Property
3. **UDP 不受影响**：TLS 仅适用于 TCP，UDP 模式下 Tls 函数调用应报错或忽略
4. **向后兼容**：不调用任何 Tls 函数时，行为与改造前完全一致
5. **编译依赖**：项目需包含 cTlsSocket.cls 及其依赖的 TLS 模块（mdTlsThunks.bas / mdTlsSodium.bas / mdTlsNative.bas）
