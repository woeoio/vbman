# After 8 Years of Development, I Open-Sourced My VB6/VBA Networking Framework — VBMAN

Hey everyone,

I've been working on something since 2017 that I think some of you might find useful, especially if you're still maintaining VB6 or VBA codebases. It's called **VBMAN** , and I just open-sourced it under GPL v3.

**Repository**: https://github.com/woeoio/vbman
**Documentation**: https://doc.vb6.pro

---

## The Problem

Let's be honest — VB6's built-in networking capabilities are basically nonexistent. Need to make an HTTP request? Host a web server? Handle WebSockets? You're on your own. The usual workaround is to write a middle layer in C# or Python, which means one more thing to deploy, one more thing to maintain, one more thing that can break.

VBMAN eliminates that middle layer. It brings modern networking directly into VB6/VBA.

---

## VBMAN: What VB6 Couldn't Do Before

VBMAN is a networking framework built specifically for VB6/VBA. It packages a bunch of modern networking capabilities into objects that VB6 can use directly. No Python, no Node.js, no middle layer — VB6 handles it on its own.

This project started from an idea and an architecture diagram in 2017, actual coding began in 2020, a major refactor happened in 2023, and it was open-sourced this June under GPL v3. The binary is free forever.

But first, let's see what it can do:

## The Highlight: HTTP Server in VB6

Yes, you read that right. You can now run a full HTTP server from VB6:

```vb
Dim Server As New cHttpServer

Private Sub Form_Load()

    ' Register the form itself as a controller
    Call Server.Router.Reg("Home", Me)

    ' Define routes — familiar pattern if you've used Express or Flask
    Call Server.Router.Add("/", "Home@Index")
    Call Server.Router.Add("/api/user/{id}", "Home@GetUser", OnlyGet)
    
    ' Need static file serving? Just specify a web root directory
    Call Server.WebRoot("C:\WebRoot")
    
    ' Start!
    If Server.Start(8080) Then
        Debug.Print "Server running at http://localhost:8080"
    Else
        MsgBox Server.LastError
    End If
End Sub

Private Sub Form_Unload(Cancel As Integer)
    Server.StopMe
End Sub
```

And the controller methods, written right in the form:
(For real projects with many endpoints, you'd use a separate class — register with `Server.Router.Reg "User", New cUserController`)

```vb
' Home page
Public Sub Index(ctx As cHttpServerContext)
    ctx.Response.Html "<h1>Hello World!</h1>"
End Sub

' RESTful route with parameter
Public Sub GetUser(ctx As cHttpServerContext)
    Dim id As String
    id = ctx.Request.RouteParams("id")

    With New cJson
        .Item("userId") = id
        .Item("name") = "John Doe"
        ctx.Response.Json .Root
    End With
End Sub
```

The context object gives you **Request**, **Response**, **Session**, **Cookies**, **Db**, and **SSE** — everything you need in one place. The API design is similar to Express.js's `req/res` pattern, adapted for VB6 syntax.

Of course, for serious projects you'd use separate class modules as controllers for cleaner code:

```vb
' cUserController.cls
Public Sub List(ctx As cHttpServerContext)
    With New cJson
        .Item("team") = Array("Alice", "Bob", "Charlie")
        .Item("total") = 3
        ctx.Response.Json .Root, 0, "Success"
    End With
End Sub
```

### What the HTTP Server Actually Supports

This isn't a toy HTTP server. It's production-ready:

- **Routing system** — Manual registration, parameter routes (`{id}`), auto-routing, HTTP method constraints, route groups, middleware
- **Session management** — Three storage backends: in-memory, filesystem, and database
- **Static file serving** — WebRoot directory hosting with lazy content caching, ETag/304 negotiation, configurable TTL
- **SSE (Server-Sent Events)** — Real-time server push, broadcast, user-targeted push, online client counting, automatic heartbeat
- **CORS** — Built-in cross-origin support for API development
- **HTTPS/TLS** — Chain-style configuration, supports PFX, PEM, and Windows certificate store. TLS 1.2/1.3
- **Connection management** — Max connections, idle timeout, request size limits, built-in cleanup timer
- **MVC pattern** — Route-controller-view architecture that scales well for larger projects

It even has a performance statistics object (`Statistics`) — request counts, status code distribution, QPS, peak connections, traffic — everything you need to build an ops monitoring dashboard.

The memory leak that existed in earlier versions has been fully resolved (v1.0.0.423 — 12 bugs fixed, 42 implicit `As New` references converted, stress-tested).

## VBMAN2: The Upgrade — With Embedded Browser

If VBMAN already caught your attention, here's more good news: its upgrade, **VBMAN2**, is already in development.

VBMAN2 is built on TwinBASIC and compiles to DLL, supporting both 32-bit and 64-bit. It's not a separate project — it's the evolution of VBMAN, designed to be a proper upgrade path.

### Already Shipped: WebView2 Object

The first component out of the gate is `cWebView2Host`. One line embeds a full Chromium browser in your VB6 form:

```vb
Dim wv As New cWebView2Host

Private Sub Form_Load()
    wv.Initialize Me.hWnd, "https://vb6.pro"
End Sub
```

That's it. Two lines and you have an Edge-powered browser running inside your form. No OCX registration, no control dragging (though you can initialize onto any control with an hWnd handle — PictureBox, Frame, etc.), no configuration.

And the VB/JS interaction is comprehensive:

```vb
' Synchronous JS execution — get return values directly
Dim title As String
title = wv.JsRun("document.title")  ' There's also a direct property: wv.DocumentTitle

' Two-way data binding between VB6 variables and web UI
wv.BindData "name", "#name-input", "value"        ' VB → UI
wv.BindUI Me, "OnInput", "#name-input", EventName:="input"  ' UI → VB

Public Sub OnInput(ByVal EventName As String, ByVal Detail As String)
    wv.SetData "name", GetValue(Detail, "value")
End Sub
```

It also packs 40+ events (navigation, scripting, mouse, keyboard, downloads, PDF printing), full CDP (Chrome DevTools Protocol) capability, cookie management, local resource mapping, and multi-host adaptation (VB6, Excel UserForm, and Access Form all supported).

### What's Coming

VBMAN2 will gradually migrate all VBMAN objects over, with most being reimplemented from scratch for better performance and richer features. The roadmap includes:

- IOCP-based high-performance networking and HTTP server
- Truly debuggable multi-threaded pool
- More database driver integrations
- A more capable AI object

VBMAN2 is not open-source, but — just like VBMAN — **the binary DLL is free to use forever**.

You can download VBMAN2 demo source code from the documentation site: https://doc.vb6.pro

## Real-World Use Cases

1. **Adding REST APIs to legacy VB6 systems** — Your 10-year-old VB6 app can now serve JSON APIs directly. No migration, no middle layer.

2. **Industrial dashboards** — Read PLC data via Modbus, push it to browsers via SSE. Real-time factory floor displays in under 100 lines of code.

3. **Internal admin tools** — VB6 as the backend, any frontend you want. Session, auth, database access all built in. One person can handle both frontend and backend.

4. **IoT gateways** — Serial port + MQTT + HTTP server in one VB6 process. Read sensors, forward messages, expose APIs.

5. **Hybrid desktop apps** — Use VBMAN2's WebView2 to embed web UIs in VB6 forms. VB handles the backend logic, HTML/CSS/JS handles the presentation, with two-way data binding connecting them.

## Quick Start

The absolute minimum to get an HTTP server running:

```vb
Dim Server As New cHttpServer

Private Sub Form_Load()
    With Server
        ' Register the form itself as controller
        .Router.Reg "Home", Me
        .Router.Add "/", "Home@Hello"
        .Start 8080
    End With
End Sub

Public Sub Hello(ctx As cHttpServerContext)
    ctx.Response.Html "<h1>Hello from VB6!</h1>"
End Sub
```

Open `http://localhost:8080` in your browser and there it is.
