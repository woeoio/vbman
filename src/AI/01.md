## 一、设计目标

实现一个通用的 AI 对象类 `cAI`，能够对接任何兼容 OpenAI API 格式的接口（如豆包、DeepSeek、OpenAI、通义千问等），开发者只需要提供 API 密钥即可使用。采用链式调用（Fluent Interface）方式配置参数，提供简洁优雅的 API。

## 二、核心设计原则

### 2.1 链式调用模式
- 所有配置方法返回 `cAI` 对象自身（`Set MethodName = Me`）
- 支持连续调用，如：`AI.ApiKey("xxx").Model("gpt-4").Temperature(0.7).Chat("你好")`
- 参考项目中 `cLayer` 和 `cHttpClient` 的实现方式

### 2.2 最小化配置
- 默认支持 OpenAI 标准 API 格式
- 只需提供 API Key 和 Base URL 即可使用
- 其他参数提供合理的默认值

### 2.3 兼容性设计
- 支持标准 OpenAI API 格式
- 支持自定义 Base URL（适配不同服务商）
- 支持自定义 Header（适配特殊认证方式）
- 支持自定义请求路径（适配不同 API 版本）

## 三、类结构设计

### 3.1 核心属性

#### 3.1.1 基础配置
- `ApiKey` (String) - API 密钥
- `BaseUrl` (String) - API 基础地址（默认：https://api.openai.com/v1）
- `ApiPath` (String) - API 路径（默认：/chat/completions）
- `Model` (String) - 模型名称（默认：gpt-3.5-turbo）
- `Timeout` (Long) - 请求超时时间（秒，默认：60）

#### 3.1.2 请求参数
- `Temperature` (Double) - 温度参数（0-2，默认：1.0）
- `MaxTokens` (Long) - 最大生成 token 数（默认：根据模型自动设置）
- `TopP` (Double) - Top-P 采样（默认：1.0）
- `FrequencyPenalty` (Double) - 频率惩罚（默认：0）
- `PresencePenalty` (Double) - 存在惩罚（默认：0）
- `Stream` (Boolean) - 是否流式响应（默认：False）
- `Stop` (Variant) - 停止序列（String 或 Array）

#### 3.1.3 消息管理
- `Messages` (Collection) - 对话消息列表
- `SystemPrompt` (String) - 系统提示词

#### 3.1.4 高级配置
- `CustomHeaders` (Dictionary) - 自定义请求头
- `Proxy` (String) - 代理地址（可选）
- `Organization` (String) - 组织 ID（OpenAI 专用）

## 四、使用示例

### 4.1 基础使用
```vb
' 方式一：创建独立对象
Dim AI As New cAI
Dim Response As String

' 最简单的使用方式
Response = AI.ApiKey("sk-xxx").Chat("你好")

' 完整配置示例
Response = AI.ApiKey("sk-xxx") _
    .Model("gpt-4") _
    .Temperature(0.7) _
    .MaxTokens(1000) _
    .System("你是一个专业的编程助手") _
    .Chat("如何实现单例模式？")

' 方式二：使用全局对象 vbman（推荐）
' 在 cVBMAN 类中添加 Public AI As New cAI
Dim Response As String
Response = VBMAN.AI.ApiKey("sk-xxx").Chat("你好")
```

### 4.2 多轮对话
```vb
Dim AI As New cAI

' 第一轮
AI.ApiKey("sk-xxx").User("我喜欢编程").Chat()

' 第二轮（自动保留上下文）
AI.User("推荐一些学习资源").Chat()

' 清空历史重新开始
AI.ClearMessages().User("新话题").Chat()
```

### 4.3 使用预设配置
```vb
Dim AI As New cAI

' 使用豆包 API
AI.Doubao("your-doubao-key").Chat("你好")

' 使用 DeepSeek API
AI.DeepSeek("your-deepseek-key").Model("deepseek-chat").Chat("你好")
```

### 4.4 流式响应

#### 方式一：使用回调函数（适合全局对象一句话使用）
```vb
Dim AI As New cAI

' 使用回调函数处理流式响应
AI.ApiKey("sk-xxx") _
    .Stream(True) _
    .Chat("写一首诗", Me, "OnStreamChunk")

' 回调方法
Private Sub OnStreamChunk(Chunk As String, IsComplete As Boolean)
    If IsComplete Then
        Debug.Print "流式响应完成"
    Else
        Debug.Print Chunk
    End If
End Sub
```

#### 方式二：使用事件（适合窗体或类中使用）
```vb
' 声明时使用 WithEvents 关键字
Dim WithEvents AI As cAI
Set AI = New cAI

' 启用流式响应并发送请求（不提供回调，自动触发事件）
AI.ApiKey("sk-xxx") _
    .Stream(True) _
    .Chat("写一首诗")

' 事件处理方法（自动触发）
Private Sub AI_OnStreamChunk(Chunk As String, IsComplete As Boolean)
    If IsComplete Then
        Debug.Print "流式响应完成"
    Else
        Debug.Print Chunk
    End If
End Sub
```

#### 方式三：全局对象 vbman 一句话使用
```vb
' 在模块或全局对象中使用
VBMAN.AI.ApiKey("sk-xxx") _
    .Stream(True) _
    .Chat("写一首诗", Me, "OnStreamChunk")

' 或者使用预设配置
VBMAN.AI.Doubao("your-key") _
    .Stream(True) _
    .Chat("你好", Me, "OnAIResponse")
```

### 4.5 获取完整响应
```vb
Dim AI As New cAI
Dim ResponseJson As cJson

Set ResponseJson = AI.ApiKey("sk-xxx").ChatJson("你好")

' 访问响应数据
Debug.Print ResponseJson.Item("choices")(0).Item("message").Item("content")
Debug.Print ResponseJson.Item("usage").Item("total_tokens")
```

## 五、技术实现要点

### 5.0 全局对象集成
- 在 `cVBMAN` 类中添加 `Public AI As New cAI`，实现全局对象访问
- 这样可以通过 `VBMAN.AI` 直接使用，无需每次创建新对象
- 支持链式调用和回调函数，方便一句话使用

### 5.1 HTTP 请求处理
- 使用项目现有的 `cHttpClient` 类进行 HTTP 请求
- `cHttpClient` 已内置支持 SSE（Server-Sent Events）客户端功能，可直接用于流式响应
- 自动设置 `Authorization: Bearer {ApiKey}` 请求头
- 自动设置 `Content-Type: application/json`
- 支持自定义请求头扩展

### 5.2 JSON 数据处理
- 使用项目现有的 `cJson` 类处理 JSON
- 自动构建 OpenAI 格式的请求体
- 自动解析响应数据

### 5.3 消息格式管理
- 维护消息列表（Collection），包含 role 和 content
- 自动添加 system、user、assistant 消息
- 支持消息历史管理（用于多轮对话）

### 5.4 错误处理
- 统一的错误处理机制
- 提供 `LastError` 属性记录错误信息
- 解析 API 返回的错误信息并抛出友好异常

### 5.5 流式响应处理
- 直接使用 `cHttpClient` 内置的 SSE 客户端功能处理流式响应
- `cHttpClient` 已实现 SSE 协议支持，自动解析流式数据块（data: {...} 格式）
- 通过 `cHttpClient` 的回调机制接收实时数据流
- **双机制支持**：
  - **回调函数机制**：如果 `Chat()` 方法提供了 `Callback` 和 `CallbackMethod` 参数，则通过 `CallByName` 调用回调函数
  - **事件机制**：如果没有提供回调函数，则通过 `RaiseEvent OnStreamChunk` 触发事件
  - 优先级：回调函数 > 事件（如果同时存在，优先使用回调函数）
- 使用 `WithEvents` 关键字声明对象以接收事件
- 回调函数签名：`Sub MethodName(Chunk As String, IsComplete As Boolean)`

### 5.6 预设配置实现
- 预设方法内部设置对应服务商的 BaseUrl 和默认参数
- 豆包：https://ark.cn-beijing.volces.com/api/v3
- DeepSeek：https://api.deepseek.com/v1
- 通义千问：https://dashscope.aliyuncs.com/api/v1

## 八、注意事项

1. **API Key 安全**：不要在代码中硬编码 API Key，建议从配置文件或环境变量读取
2. **错误处理**：所有 API 调用都可能失败，需要完善的错误处理机制
3. **超时设置**：根据模型和网络情况合理设置超时时间
4. **Token 限制**：注意不同模型的 token 限制，避免超出限制
5. **成本控制**：流式响应和长文本会消耗更多 token，注意成本控制
6. **兼容性**：不同服务商的 API 可能有细微差异，需要测试验证
