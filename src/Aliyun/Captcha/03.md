## 三、HTTP 请求构建详解

### 3.1 API 参数配置

```go
params := &openapi.Params{
    Action:      tea.String("VerifyIntelligentCaptcha"),  // API 名称
    Version:     tea.String("2023-03-05"),                // API 版本
    Protocol:    tea.String("HTTPS"),                     // 协议
    Pathname:    tea.String("/"),                         // 路径
    Method:      tea.String("POST"),                      // 方法
    AuthType:    tea.String("AK"),                        // 认证类型
    Style:       tea.String("RPC"),                       // 请求风格
    ReqBodyType: tea.String("formData"),                  // 请求体类型
    BodyType:    tea.String("json"),                      // 响应体类型
}
```

### 3.2 完整 HTTP 请求构建流程（RPC 风格）

#### 步骤 1：构建基础 Query 参数

```
URL: https://captcha.cn-shanghai.aliyuncs.com/
```

**系统级参数（自动添加）：**

| 参数名           | 值                       | 说明                          |
| ---------------- | ------------------------ | ----------------------------- |
| Action           | VerifyIntelligentCaptcha | API 名称                      |
| Format           | json                     | 响应格式                      |
| Version          | 2023-03-05               | API 版本                      |
| Timestamp        | 2024-01-15T08:30:00Z     | ISO8601 格式 UTC 时间         |
| SignatureNonce   | xxxxxxxx                 | 唯一随机数（UUID/随机字符串） |
| AccessKeyId      | LTAIxxxxx                | 你的 AccessKey ID             |
| SignatureMethod  | HMAC-SHA1                | 签名算法                      |
| SignatureVersion | 1.0                      | 签名版本                      |

#### 步骤 2：构建业务参数（Body）

```
Content-Type: application/x-www-form-urlencoded

Body: CaptchaVerifyParam=xxx&SceneId=xxx
```

**参数编码规则：**

- 参数名和值都需要进行 URL 编码（RFC3986）
- 空格编码为 `%20`，不是 `+`
- 特殊字符：`+` → `%2B`，`*` → `%2A`，`%7E` → `~`

#### 步骤 3：构建 HTTP Headers

```
Host: captcha.cn-shanghai.aliyuncs.com
x-acs-version: 2023-03-05
x-acs-action: VerifyIntelligentCaptcha
user-agent: AlibabaCloud (Windows;) Go/1.18
Content-Type: application/x-www-form-urlencoded
```

## 四、签名算法详解（关键）

### 4.1 RPC V1 签名算法（HMAC-SHA1）

这是验证码 API 使用的签名方式。

#### 签名流程

```
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 构造待签名字符串 (StringToSign)                      │
├─────────────────────────────────────────────────────────────┤
│  HTTPMethod + "&" +                                         │
│  percentEncode("/") + "&" +                                 │
│  percentEncode(CanonicalizedQueryString)                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 2: 计算 HMAC-SHA1 签名                                  │
├─────────────────────────────────────────────────────────────┤
│  Signature = Base64( HMAC-SHA1( AccessKeySecret + "&",      │
│                                 StringToSign ) )            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 添加签名到请求                                       │
├─────────────────────────────────────────────────────────────┤
│  Query["Signature"] = URL_ENCODE(Signature)                 │
└─────────────────────────────────────────────────────────────┘
```

#### 详细步骤

**Step 1: 构造规范化查询字符串（CanonicalizedQueryString）**

```
1. 提取所有参数（Query + Body 中的参数）
2. 按参数名字典序（ASCII）升序排列
3. 对每个参数名和值进行 URL 编码
4. 用 "=" 连接编码后的参数名和值
5. 用 "&" 连接所有参数对
```

**示例：**

原始参数：

```
Action=VerifyIntelligentCaptcha
CaptchaVerifyParam=前端参数
SceneId=场景ID
Timestamp=2024-01-15T08:30:00Z
SignatureNonce=123456
AccessKeyId=LTAIxxxxx
SignatureMethod=HMAC-SHA1
SignatureVersion=1.0
Format=json
Version=2023-03-05
```

排序后编码：

```
AccessKeyId=LTAIxxxxx
Action=VerifyIntelligentCaptcha
CaptchaVerifyParam=%E5%89%8D%E7%AB%AF%E5%8F%82%E6%95%B0
Format=json
SceneId=%E5%9C%BA%E6%99%AFID
SignatureMethod=HMAC-SHA1
SignatureNonce=123456
SignatureVersion=1.0
Timestamp=2024-01-15T08%3A30%3A00Z
Version=2023-03-05
```

拼接后：

```
AccessKeyId=LTAIxxxxx&Action=VerifyIntelligentCaptcha&CaptchaVerifyParam=%E5%89%8D%E7%AB%AF%E5%8F%82%E6%95%B0&Format=json&SceneId=%E5%9C%BA%E6%99%AFID&SignatureMethod=HMAC-SHA1&SignatureNonce=123456&SignatureVersion=1.0&Timestamp=2024-01-15T08%3A30%3A00Z&Version=2023-03-05
```

**Step 2: 构造待签名字符串**

```
StringToSign =
    "POST" + "&" +
    percentEncode("/") + "&" +
    percentEncode(上面生成的规范化查询字符串)

结果：
POST&%2F&AccessKeyId%3DLTAIxxxxx%26Action%3DVerifyIntelligentCaptcha%26...
```

**Step 3: 计算签名**

```
AccessKeySecret = "你的AccessKey Secret"
Key = AccessKeySecret + "&"  // 注意：末尾要加 &

Signature = Base64( HMAC_SHA1(Key, StringToSign) )
```

**Step 4: 添加到请求**

```
最终的 Query String = 原来的参数 + "&Signature=" + URL_ENCODE(Signature)
```

### 4.2 URL 编码规则（RFC3986）

```vb6
' VB6 实现示例
Function PercentEncode(ByVal str As String) As String
    Dim i As Long, c As String
    Dim result As String

    For i = 1 To Len(str)
        c = Mid(str, i, 1)
        Select Case c
            Case "A" To "Z", "a" To "z", "0" To "9", "-", "_", ".", "~"
                result = result & c
            Case " "
                result = result & "%20"
            Case Else
                result = result & "%" & Hex(Asc(c))
        End Select
    Next i

    PercentEncode = result
End Function
```

### 6.1 成功响应

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
    "RequestId": "89xxxxx-xxxx-xxxx-xxxx-xxxxxxxx",
    "Code": "Success",
    "Success": true,
    "Result": {
        "VerifyResult": true,
        "VerifyCode": "100"
    }

```

### 6.2 响应字段说明

| 字段                | 类型    | 说明                         |
| ------------------- | ------- | ---------------------------- |
| RequestId           | String  | 请求唯一标识                 |
| Code                | String  | 业务状态码，Success 表示成功 |
| Success             | Boolean | 请求是否成功                 |
| Result.VerifyResult | Boolean | 验证码验证结果               |
| Result.VerifyCode   | String  | 原因代码，100 表示通过       |

**VerifyCode 说明：**

- `100` - 验证通过
- `200` - 验证失败（其他原因）
- `其他` - 具体错误码

## 七、VB6 实现要点

### 7.1 核心流程

```vb6
Sub VerifyCaptcha(ByVal accessKeyId As String, _
                  ByVal accessKeySecret As String, _
                  ByVal sceneId As String, _
                  ByVal captchaVerifyParam As String)

    Dim endpoint As String
    endpoint = "captcha.cn-shanghai.aliyuncs.com"

    ' 1. 构建基础参数
    Dim params As Dictionary
    Set params = New Dictionary
    params.Add "Action", "VerifyIntelligentCaptcha"
    params.Add "Version", "2023-03-05"
    params.Add "Format", "json"
    params.Add "AccessKeyId", accessKeyId
    params.Add "SignatureMethod", "HMAC-SHA1"
    params.Add "SignatureVersion", "1.0"
    params.Add "Timestamp", GetISOTimestamp()
    params.Add "SignatureNonce", GetUUID()

    ' 2. 构建业务参数（放 Body 中）
    Dim bodyParams As Dictionary
    Set bodyParams = New Dictionary
    bodyParams.Add "CaptchaVerifyParam", captchaVerifyParam
    bodyParams.Add "SceneId", sceneId

    ' 3. 计算签名（合并 Query 和 Body 参数）
    Dim signature As String
    signature = CalculateSignature(params, bodyParams, accessKeySecret)
    params.Add "Signature", signature

    ' 4. 构建请求 URL
    Dim url As String
    url = "https://" & endpoint & "/?" & BuildQueryString(params)

    ' 5. 发送 POST 请求
    Dim body As String
    body = BuildQueryString(bodyParams)

    Dim response As String
    response = HttpPost(url, body)

    ' 6. 解析响应
    ParseResponse(response)
End Sub
```

### 7.2 需要实现的辅助函数

| 函数                   | 说明                       |
| ---------------------- | -------------------------- |
| `GetISOTimestamp()`    | 生成 ISO8601 格式 UTC 时间 |
| `GetUUID()`            | 生成唯一随机字符串         |
| `PercentEncode()`      | RFC3986 URL 编码           |
| `CalculateSignature()` | HMAC-SHA1 签名计算         |
| `BuildQueryString()`   | 构建查询字符串             |
| `HttpPost()`           | HTTPS POST 请求            |
| `ParseResponse()`      | JSON 响应解析              |

### 7.3 时间戳格式

```vb6
Function GetISOTimestamp() As String
    ' 格式: 2024-01-15T08:30:00Z
    Dim utc As Date
    utc = GetUTCTime()  ' 需要转换为 UTC 时间

    GetISOTimestamp = Format(utc, "yyyy-mm-ddThh:mm:ss") & "Z"
End Function
```

### 7.4 注意事项

1. **时间同步**：确保服务器时间与阿里云时间同步（误差不超过 15 分钟）
2. **HTTPS**：必须使用 HTTPS，不支持 HTTP
3. **编码**：所有参数必须经过 URL 编码
4. **大小写**：参数名区分大小写
5. **重复请求**：SignatureNonce 必须唯一，避免重放攻击
