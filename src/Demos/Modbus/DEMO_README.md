# Modbus 演示程序使用说明

本目录包含完整的 Modbus 主站和从站演示程序,支持 TCP 和 RTU 两种协议模式。

## 程序说明

### 1. 主站演示程序 (Master)

**位置**: `Master/MasterDemo.vbp`

**功能**:
- Modbus 主站功能演示
- 支持 TCP 和 RTU 两种连接模式
- 完整的读写操作测试
- 从站数据状态实时查看

**特性**:
- 图形化配置界面
- 连接状态实时显示
- 读写操作按钮
- 操作日志记录
- 从站数据刷新功能

**支持的读写操作**:
- 读线圈 (FC 01)
- 读离散输入 (FC 02)
- 读保持寄存器 (FC 03)
- 读输入寄存器 (FC 04)
- 写单个线圈 (FC 05)
- 写单个寄存器 (FC 06)
- 写多个线圈 (FC 15)
- 写多个寄存器 (FC 16)

### 2. 从站演示程序 (Slave)

**位置**: `Slave/SlaveDemo.vbp`

**功能**:
- Modbus 从站功能演示
- 支持 TCP 和 RTU 两种服务模式
- 数据管理和设置
- 主站请求实时监控
- 可配置监听地址（BindAddress）

**特性**:
- 服务器启动/停止控制
- 线圈、离散输入、寄存器数据管理
- 客户端连接状态显示
- 读写请求事件记录
- 测试数据自动初始化
- 支持配置监听地址（127.0.0.1、0.0.0.0 或指定 IP）

**支持的数据类型**:
- 线圈 (Coils): 0-2000
- 离散输入 (Discrete Inputs): 0-2000
- 保持寄存器 (Holding Registers): 0-125
- 输入寄存器 (Input Registers): 0-125

## 使用步骤

### TCP 模式测试

#### 1. 启动从站程序

1. 打开 `Slave/SlaveDemo.vbp` 项目
2. 选择 "TCP" 协议模式
3. 设置监听端口 (默认 502)
4. 设置从站ID (默认 1)
5. (可选) 设置监听地址：
   - `127.0.0.1` - 仅允许本地连接（推荐测试使用）
   - `0.0.0.0` - 监听所有网络接口（默认）
   - `192.168.1.100` - 监听指定网卡
6. 点击"启动服务器"
7. 确认状态显示为"运行中"

#### 2. 启动主站程序

1. 打开 `Master/MasterDemo.vbp` 项目
2. 选择 "TCP" 协议模式
3. 设置主机地址 (127.0.0.1 表示本机)
4. 设置端口 (与从站一致,默认 502)
5. 设置从站ID (与从站一致,默认 1)
6. 点击"连接"
7. 确认连接成功

#### 3. 执行测试操作

**读取测试**:
1. 设置读取地址和数量
2. 点击相应读取按钮
3. 查看日志中的返回结果

**写入测试**:
1. 设置写入地址和值
2. 点击相应写入按钮
3. 确认写入成功
4. 刷新从站数据查看变更

**刷新从站数据**:
1. 在从站程序中修改数据
2. 在主站程序中点击"刷新从站数据"
3. 查看最新的从站状态

### RTU 模式测试

#### 准备工作

RTU 模式需要串口设备或虚拟串口对。可以使用以下工具:
- 物理串口: 使用两个物理串口,一个连接主站,一个连接从站
- 虚拟串口: 使用虚拟串口软件(如 com0com)创建串口对

#### 1. 启动从站程序

1. 打开 `Slave/SlaveDemo.vbp` 项目
2. 选择 "RTU" 协议模式
3. 设置串口参数 (串口、波特率、数据位、校验位、停止位)
4. 设置从站ID (默认 1)
5. 点击"启动服务器"
6. 确认状态显示为"运行中"

#### 2. 启动主站程序

1. 打开 `Master/MasterDemo.vbp` 项目
2. 选择 "RTU" 协议模式
3. 设置串口参数 (必须与从站一致)
4. 设置从站ID (与从站一致)
5. 点击"连接"
6. 确认连接成功

#### 3. 执行测试操作

RTU 模式的测试操作与 TCP 模式完全相同,参考上面的 TCP 测试步骤。

## 常见问题

### Q1: TCP 连接失败?

**解决方案**:
- 检查从站是否已启动
- 检查端口是否被占用 (使用 `netstat -ano` 查看)
- 检查防火墙设置
- 确认主机地址正确
- 检查从站监听地址设置（BindAddress）
- 确认监听地址允许当前客户端连接

### Q2: RTU 连接失败?

**解决方案**:
- 检查串口是否存在
- 检查串口是否被其他程序占用
- 确认串口参数一致 (波特率、数据位、校验位、停止位)
- 尝试使用虚拟串口软件测试

### Q3: 读取返回错误数据?

**解决方案**:
- 检查从站ID是否匹配
- 检查地址范围是否有效
- 确认从站支持该功能码
- 查看日志中的详细错误信息

### Q4: 写入操作无效果?

**解决方案**:
- 检查地址是否正确
- 确认值在有效范围内
- 检查从站数据是否可写
- 在从站程序中手动验证数据

### Q5: RTU CRC 错误?

**解决方案**:
- 确认串口参数完全一致
- 检查串口线连接
- 增加超时时间
- 检查波特率设置
