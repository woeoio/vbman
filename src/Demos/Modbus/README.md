# Modbus 测试示例

本目录包含 Modbus 类的完整测试示例，包括图形界面测试工具和命令行测试代码。

## 文件说明

### 1. FModbusTest.frm - 图形界面测试工具

完整的 Windows 表单应用程序，提供可视化的 Modbus 测试界面。

**功能特性：**
- 支持 RTU 和 TCP 两种协议模式
- 图形化配置连接参数
- 完整的读取操作测试（线圈、离散输入、保持寄存器、输入寄存器）
- 完整的写入操作测试（单个/多个线圈、单个/多个寄存器）
- 实时操作日志显示
- 事件处理（连接、断开、错误）

**使用方法：**
1. 在 VB6 中打开表单文件
2. 选择协议类型（RTU 或 TCP）
3. 配置连接参数
4. 点击"连接"按钮
5. 使用各种读取/写入按钮进行测试
6. 查看日志窗口了解操作结果

### 2. ModbusTest.bas - 命令行测试模块

提供完整的命令行测试函数，适合在立即窗口或代码中调用。

**主要函数：**
- `TestModbusTCP()` - TCP 模式完整测试
- `TestModbusRTU()` - RTU 模式完整测试
- `TestModbusAll()` - 综合测试（包含所有功能）
- `QuickTest()` - 快速测试示例

**使用方法：**
```vb
' 在立即窗口中执行
TestModbusTCP    ' 测试 TCP 模式
TestModbusRTU    ' 测试 RTU 模式
QuickTest         ' 快速测试
```

### 3. demo.bas - 简单示例代码

提供最基础的使用示例，适合快速参考。

**包含示例：**
- `DemoTCP()` - TCP 模式简单示例
- `DemoRTU()` - RTU 模式简单示例

## 测试步骤

### TCP 模式测试

1. **准备 Modbus TCP 服务器**
   - 可以使用 Modbus 模拟器（如 Modbus Poll/Slave）
   - 或连接实际的 Modbus TCP 设备

2. **配置连接参数**
   - 主机地址：设备 IP 地址（如 192.168.1.100）
   - 端口：默认 502
   - 从站ID：设备地址（通常为 1）

3. **执行测试**
   - 使用图形界面工具或调用测试函数
   - 测试各种读取/写入操作

### RTU 模式测试

1. **准备串口设备**
   - 连接 Modbus RTU 设备到串口
   - 或使用虚拟串口工具进行测试

2. **配置串口参数**
   - 串口：COM1, COM2 等
   - 波特率：9600（常用）
   - 数据位：8
   - 校验位：N（无校验）、E（偶校验）、O（奇校验）
   - 停止位：1 或 2

3. **执行测试**
   - 使用图形界面工具或调用测试函数
   - 测试各种读取/写入操作

## 功能码测试

### 读取操作

- **读线圈 (FC 01)**
  - 地址范围：0-65535
  - 数量限制：1-2000
  - 返回：Boolean 数组

- **读离散输入 (FC 02)**
  - 地址范围：0-65535
  - 数量限制：1-2000
  - 返回：Boolean 数组

- **读保持寄存器 (FC 03)**
  - 地址范围：0-65535
  - 数量限制：1-125
  - 返回：Integer 数组（16位）

- **读输入寄存器 (FC 04)**
  - 地址范围：0-65535
  - 数量限制：1-125
  - 返回：Integer 数组（16位）

### 写入操作

- **写单个线圈 (FC 05)**
  - 地址范围：0-65535
  - 值：True/False

- **写单个寄存器 (FC 06)**
  - 地址范围：0-65535
  - 值：Integer（-32768 到 32767）

- **写多个线圈 (FC 15)**
  - 地址范围：0-65535
  - 数量限制：1-2000
  - 值：Boolean 数组

- **写多个寄存器 (FC 16)**
  - 地址范围：0-65535
  - 数量限制：1-125
  - 值：Integer 数组

## 注意事项

1. **连接状态**
   - 确保设备已正确连接并配置
   - 检查网络连接（TCP）或串口连接（RTU）

2. **地址范围**
   - Modbus 地址从 0 开始
   - 某些设备可能使用 1-based 地址，需要相应调整

3. **超时设置**
   - 默认超时为 1000ms
   - 可以通过 `ResponseTimeout` 属性调整

4. **错误处理**
   - 所有操作都包含错误处理
   - 错误信息会显示在日志中或通过事件通知

5. **线程安全**
   - Modbus 操作是同步的
   - 在 UI 线程中执行长时间操作可能阻塞界面

## 常见问题

**Q: 连接失败怎么办？**
- 检查设备是否在线
- 检查 IP 地址和端口（TCP）
- 检查串口是否被占用（RTU）
- 检查串口参数是否正确（RTU）

**Q: 读取返回空数组？**
- 检查从站ID是否正确
- 检查地址范围是否有效
- 检查设备是否支持该功能码

**Q: 写入失败？**
- 检查寄存器/线圈是否可写
- 检查地址是否在允许范围内
- 检查值是否在有效范围内
