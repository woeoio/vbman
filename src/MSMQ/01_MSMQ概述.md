# MSMQ (Microsoft Message Queuing) 概述

## 什么是 MSMQ？

MSMQ（Microsoft Message Queuing，微软消息队列）是微软提供的**系统级消息队列中间件**，允许应用程序通过消息队列进行异步通信。

## 核心特点

| 特性 | 说明 |
|------|------|
| 异步通信 | 发送方和接收方无需同时在线 |
| 可靠传输 | 消息持久化存储，保证不丢失 |
| 离线支持 | 接收方离线时消息自动排队 |
| 事务支持 | 支持事务性消息处理 |
| 优先级 | 可设置消息优先级 |
| 路由 | 支持跨网络路由消息 |

## 适用场景

1. **高并发处理** - 削峰填谷，缓解服务器压力（如文章提到的 2000 并发场景）
2. **异步处理** - 耗时操作后台处理（发送邮件、生成报表）
3. **系统解耦** - 不同系统间通过消息通信，降低耦合度
4. **负载均衡** - 多个消费者并发处理消息
5. **离线处理** - 网络不稳定环境下的可靠传输

## 队列类型

### 1. 公共队列 (Public Queue)
- 在 Active Directory 中注册
- 可通过网络名称访问
- 需要域环境支持
- 路径格式：`队列名`

### 2. 专用队列 (Private Queue)
- 不在 Active Directory 中注册
- 仅在本地计算机可用
- 无需域环境
- 路径格式：`private$\队列名`

### 3. 日志队列 (Journal Queue)
- 系统级队列，记录所有消息副本
- 用于审计和故障排查
- 路径格式：`队列名\Journal$`

### 4. 死信队列 (Dead-Letter Queue)
- 存储无法送达的消息
- 分为系统死信队列和事务死信队列
- 路径：`deadletter$` 或 `xactdeadletter$`

## 消息类型

### 普通消息
- 标准的消息传递方式
- 支持持久化和非持久化

### 事务性消息
- 参与 MSMQ 事务
- 保证原子性操作
- 需要事务性队列

### 确认消息
- 通知发送方消息状态
- 可追踪消息是否送达、读取

## 系统要求
| 操作系统 | 支持版本 |
|---------|---------|
| Windows 11 | 专业版/企业版/教育版 |
| Windows 10 | 专业版/企业版/教育版 |
| Windows Server | 2019/2016/2012 R2 等 |

**注意**：Windows 家庭版可能不支持 MSMQ 或功能受限。

## 限制

- **仅支持 Windows 操作系统**
- 消息大小默认限制为 4MB
- 队列存储受磁盘空间限制
- 跨域访问需要额外配置
