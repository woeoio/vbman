# MSMQ 高级主题

## 一、直接格式名（Direct Format Names）
直接格式名用于在不访问目录服务的情况下引用公共或专用队列，适用于以下场景：
- 直接发送消息到计算机
- 通过 Internet 发送消息
- 跨林（Forest）边界发送消息
- 在域模式、工作组模式或离线模式下操作

### 格式名语法
```
' 公共队列
DIRECT=AddressSpecification\QueueName

' 专用队列
DIRECT=AddressSpecification\PRIVATE$\QueueName

' 日志队列
DIRECT=AddressSpecification\QueueName;JOURNAL

' 死信队列
DIRECT=OS:ComputerName\SYSTEM$;DEADLETTER

' 事务死信队列
DIRECT=OS:ComputerName\SYSTEM$;DEADXACT
```

### 地址规范
| 协议 | 说明 | 示例 |
|------|------|------|
| TCP | 面向连接的 TCP/IP | `DIRECT=TCP:192.168.1.100\MyQueue` |
| OS | 使用原生计算机命名约定 | `DIRECT=OS:Server01\MyQueue` |
| HTTP | HTTP 传输 | `DIRECT=HTTP://server/msmq/MyQueue` |
| HTTPS | 安全 HTTP (SSL) | `DIRECT=HTTPS://server/msmq/MyQueue` |

### VB6 中使用直接格式名
```vb
Dim qinfo As New MSMQQueueInfo

' 通过 IP 访问远程队列
qinfo.FormatName = "DIRECT=TCP:192.168.1.100\private$\RemoteQueue"

' 通过计算机名访问
qinfo.FormatName = "DIRECT=OS:Server01\private$\MyQueue"

' 使用句点表示本地计算机
qinfo.FormatName = "DIRECT=OS:.\private$\LocalQueue"

Set queue = qinfo.Open(MQ_SEND_ACCESS, MQ_DENY_NONE)
```

### 直接格式名的安全限制
使用直接格式名打开远程队列时，MSMQ 不会查询目录服务，这会失去一些安全功能：

1. **访问控制** - 无法使用 MSMQ 的访问控制功能
2. **身份验证** - 消息可以验证内容未被篡改，但验证发送者身份需要外部证书
3. **加密** - MSMQ 无法自动加密直接格式名的消息

**建议**：跨信任边界发送消息时，设置 `SenderIDType = MQMSG_SENDERID_TYPE_NONE`

## 二、毒消息处理（Poison Messages）
毒消息是指已超过最大传递尝试次数的消息。这种情况通常发生在：
- 应用程序读取消息后处理出错
- 事务中止，消息返回队列
- 问题未解决，形成无限循环

### 毒消息特征
- 阻碍队列中其他有效消息的处理
- 可能导致整个消息处理系统停滞
- 需要特殊机制来处理

### MSMQ 提供的毒消息处理功能
#### 1. 中止计数（Abort Count）
```vb
' 获取消息在当前队列中的中止次数
Dim abortCount As Long
abortCount = msg.AbortCount  ' PROPID_M_ABORT_COUNT

' 注意：消息移动到子队列时，此计数会重置
```
#### 2. 移动计数（Move Count）
```vb
' 获取消息生命周期中的总中止次数
Dim moveCount As Long
moveCount = msg.MoveCount  ' PROPID_M_MOVE_COUNT

' 此计数在移动到子队列时不会重置
```
#### 3. 毒消息处理策略
```vb
' 策略1：移动到重试队列
If msg.AbortCount > 3 Then
    ' 移动到重试队列，稍后处理
    MQMoveMessage msg.Handle, retryQueueHandle, 0
End If

' 策略2：标记为拒绝（发送 Nack）
If msg.AbortCount > 5 Then
    ' 消息被移除，发送方收到 MQMSG_CLASS_NACK_RECEIVE_REJECTED
    MQMarkMessageRejected msg.Handle
End If
```
### VB6 毒消息处理示例
```vb
Private Sub ProcessMessage(ByVal msg As MSMQMessage)
    On Error GoTo ErrorHandler
    
    ' 检查中止次数
    If msg.AbortCount > 3 Then
        ' 移动到毒消息队列
        MoveToPoisonQueue msg
        Exit Sub
    End If
    
    ' 正常处理消息
    ' ...
    
    Exit Sub
    
ErrorHandler:
    ' 出错时事务回滚，增加中止计数
    Err.Raise vbObjectError, "ProcessMessage", "处理失败"
End Sub
```

## 三、MSMQ 事务详解
MSMQ 使用两个独立的事务：
1. **发送事务** - 将消息发送到目标队列
2. **接收事务** - 从队列检索消息

### 事务确认过程
当在事务中发送多个消息时，MSMQ 使用内部 **EOD（Exactly-Once-Delivery）** 协议保证消息：
- 只传递一次
- 按顺序传递

每个事务性消息附加的信息包括：
- **序列标识符** - 标识单个事务中的消息流
- **序列号** - 标识流中的消息位置
- **前一个号** - 标识前一个消息

### 事务规则
```
❌ 不允许：在一个事务中既发送又接收同一条消息
✅ 允许：在一个事务中从队列A接收，再发送到队列B
```

## 四、消息确认机制
### 确认类型
| 确认类型 | 说明 | 常量 |
|---------|------|------|
| 无确认 | 不发送确认 | MQMSG_ACKNOWLEDGMENT_NONE |
| 到达队列 | 消息到达目标队列时确认 | MQMSG_ACKNOWLEDGMENT_FULL_REACH_QUEUE |
| 被接收 | 消息被应用程序读取时确认 | MQMSG_ACKNOWLEDGMENT_FULL_RECEIVE |
| 否定确认 | 失败时发送 Nack | MQMSG_ACKNOWLEDGMENT_NACK_REACH_QUEUE |

### VB6 设置确认
```vb
Dim msg As New MSMQMessage

' 请求到达队列确认
msg.Acknowledge = MQMSG_ACKNOWLEDGMENT_FULL_REACH_QUEUE

' 同时设置管理队列接收确认
msg.AdminQueueInfo = adminQueueInfo
```

## 五、HTTP/HTTPS 支持（MSMQ 3.0+）
MSMQ 3.0 引入了通过 HTTP/HTTPS 协议发送消息的功能。
### 特点
- 可穿越防火墙
- 支持 Internet 消息传递
- 使用 IIS 作为消息接收端点

### 限制
- 只能发送，不能通过 HTTP 接收消息
- HTTPS 需要 SSL 证书配置
- 性能比原生 MSMQ 协议低
### 路径格式
```
DIRECT=HTTP://192.168.1.1/msmq/MyQueue
DIRECT=HTTP://server.domain.com/msmq/MyQueue
DIRECT=HTTPS://secure.server.com/msmq/MyQueue
```

## 六、子队列（Subqueues，MSMQ 4.0+）
MSMQ 4.0 引入了子队列概念，用于临时存储消息。
### 用途
- 存储正在处理的消息
- 实现优先级处理逻辑
- 临时存放毒消息
### 子队列类型
```
队列名\$priojournal   ' 优先级日志
队列名\deadletter      ' 死信子队列
队列名\retry           ' 自定义重试队列
```

## 七、多元素格式名（Multiple-Element Format Names）
用于一次向多个队列发送消息。
```vb
' 同时发送到多个队列
Dim formatName As String
formatName = "PUBLIC=Queue1GUID;PUBLIC=Queue2GUID;PUBLIC=Queue3GUID"

Set qinfo = New MSMQQueueInfo
qinfo.FormatName = formatName
Set queue = qinfo.Open(MQ_SEND_ACCESS, MQ_DENY_NONE)
```
---

使用分发列表一次向多个目标发送相同消息。

## 九、性能优化建议
### 发送优化
1. **批量发送** - 减少网络往返次数
2. **Express 消息** - 不需要持久化时使用 MQMSG_DELIVERY_EXPRESS
3. **异步发送** - 使用直接格式名减少目录服务查询

### 接收优化
1. **批量接收** - 一次读取多条消息
2. **Peek 优先** - 先查看再决定是否接收
3. **多线程/多实例** - 并行处理消息

### 队列设计
1. **避免单个大队列** - 按业务拆分多个队列
2. **设置队列配额** - 防止磁盘空间耗尽
3. **定期清理日志** - 删除过期日志消息
