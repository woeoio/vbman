# MSMQ 最佳实践指南

## 一、设计原则

### 1.1 队列命名规范
```
建议格式：[系统].[模块].[类型].[名称]

示例：
- ERP.Order.Create      ' ERP系统订单创建队列
- CRM.Customer.Update   ' CRM系统客户更新队列
- SYS.Log.Error         ' 系统错误日志队列
```

### 1.2 队列分类策略
| 队列类型 | 用途 | 示例 |
|---------|------|------|
| 业务队列 | 核心业务处理 | Order.Create, Payment.Process |
| 重试队列 | 失败消息重试 | Order.Retry, Payment.Retry |
| 死信队列 | 无法处理的消息 | Order.DeadLetter |
| 日志队列 | 审计和监控 | System.Log.Audit |

```vb

Private Const MAX_MESSAGE_SIZE = 4 * 1024 * 1024  ' 4MB

Private Function CheckMessageSize(ByRef msg As MSMQMessage) As Boolean
    ' 检查消息大小
    Dim size As Long
    On Error Resume Next
    size = Len(msg.Body)
    
    If size > MAX_MESSAGE_SIZE Then
        ' 分割大消息或存储到外部
        CheckMessageSize = False
    Else
        CheckMessageSize = True
    End If
End Function
```

## 三、错误处理与重试机制

### 3.1 分层重试策略
```vb
Public Sub ProcessWithRetry(ByRef msg As cMSMQMessage)
    Const MAX_RETRIES = 3
    Const RETRY_DELAYS = "1000,5000,30000"  ' 1秒, 5秒, 30秒
    
    Dim retryCount As Long
    Dim delays() As String
    delays = Split(RETRY_DELAYS, ",")
    
    retryCount = msg.AppSpecific  ' 使用 AppSpecific 存储重试次数
    
    On Error GoTo ErrorHandler
    
    ' 尝试处理
    ProcessMessage msg
    Exit Sub
    
ErrorHandler:
    retryCount = retryCount + 1
    
    If retryCount <= MAX_RETRIES Then
        ' 延迟后重试
        Sleep CLng(delays(retryCount - 1))
        msg.AppSpecific = retryCount
        ' 重新发送消息到重试队列
        SendToRetryQueue msg
    Else
        ' 超过重试次数，发送到死信队列
        SendToDeadLetterQueue msg, Err.Description
    End If
End Sub
```

### 3.2 死信队列处理
```vb
Public Sub HandleDeadLetter(ByRef msg As cMSMQMessage)
    ' 1. 记录日志
    LogError "DeadLetter", msg.MessageId, msg.Body, msg.CorrelationId
    
    ' 2. 发送告警通知
    SendAlert "Message moved to dead letter queue", msg
    
    ' 3. 可选：人工介入处理
    ' 保存到数据库等待人工处理
    SaveToManualProcessQueue msg
End Sub
```

### 4.1 批量处理
```vb
Public Sub BatchReceive(ByVal batchSize As Long)
    Dim messages As New Collection
    Dim msg As cMSMQMessage
    Dim i As Long
    
    ' 批量读取
    For i = 1 To batchSize
        Set msg = m_Client.ReceiveMessage(100)  ' 100ms超时
        If msg Is Nothing Then Exit For
        messages.Add msg
    Next i
    
    ' 批量处理
    If messages.Count > 0 Then
        ProcessBatch messages
    End If
End Sub
```

## 五、监控与运维

### 5.1 队列健康检查
```vb
Public Function CheckQueueHealth(ByVal queueName As String) As String
    Dim mgr As New cMSMQManager
    Dim info As Object
    Dim health As String
    
    If mgr.QueueExists(queueName) Then
        Set info = mgr.GetQueueInfo(queueName)
        
        Dim msgCount As Long
        msgCount = info("MessageCount")
        
        Select Case msgCount
            Case 0 To 100
                health = "健康"
            Case 101 To 1000
                health = "警告：消息积压"
            Case Else
                health = "严重：消息大量积压"
        End Select
        
        CheckQueueHealth = health & " (消息数: " & msgCount & ")"
    Else
        CheckQueueHealth = "队列不存在"
    End If
End Function
```

### 6.2 消息加密建议
```vb
Public Function EncryptMessage( _
    ByVal plainText As String, _
    ByVal key As String _
) As String
    ' 使用简单的 XOR 加密（生产环境请使用更强的加密）
    Dim i As Long
    Dim result As String
    
    For i = 1 To Len(plainText)
        result = result & Chr(Asc(Mid(plainText, i, 1)) Xor Asc(Mid(key, (i - 1) Mod Len(key) + 1, 1)))
    Next i
    
    EncryptMessage = result
End Function
```

## 七、常见问题与解决方案

### 7.1 消息丢失
**原因分析：**
- 使用了 Express 消息且系统崩溃
- 队列已满
- 事务未正确提交

**解决方案：**
- 重要消息使用 Recoverable 传递方式
- 监控队列大小，及时扩容
- 完善事务处理逻辑

### 7.2 消息顺序错乱
**原因分析：**
- 多个发送者并发发送
- 优先级设置不当
- 网络延迟不一致

**解决方案：**
- 使用事务保证同一发送者的消息顺序
- 合理使用消息优先级
- 在应用层增加序列号，自行排序

### 7.3 队列访问被拒绝
**原因分析：**
- 权限配置不当
- 队列被其他进程独占

**解决方案：**
- 检查并配置正确的队列 ACL
- 使用 MQ_DENY_NONE 共享模式

### 7.4 性能下降
**排查步骤：**
1. 检查队列中消息数量
2. 检查磁盘空间
3. 检查网络延迟
4. 检查日志队列大小
5. 检查死信队列积压
