# 需求

参考 demos\Toast\Form1.frm 的示例，把
demos\Toast\FToastCenter.frm
demos\Toast\FToastDrawer.frm
统一封装为 demos\Toast\cToast.cls 这个对象，对外使用。

2个 frm 已经把 toast 需求完整的提供了各种函数能力了，你只需要统一封装细微差别，屏蔽差异，对外统一调用函数，就可以了，具体来说就是：(以下假设 Toast 是实例变量)

- 默认不带任何参数，就是屏幕居中显示，比如 Toast.Show("你好",[3000],[提示])
- 其他函数和2个frm里面的保持一致性。
- 在内部完成差异屏蔽，比如，用户调用

```vb
with new cToast
    .Pos Center
    .Show "hi"
end with
```

此时应该调用FToastCenter，也就是你需要在内部判断是否居中的位置，有个函数 IsCenter 可以用。好像也就这一个Pos才需要区分，其他成员都可以直接复用

- 在完成所有可用frm的成员封装之后，你应该继续扩展一些辅助能力。
- 比如，你需要在cToast内部增加 集合对象，用于收集用户show的每一个弹窗对象实例。
- 你可能需要2个集合对象，分别存储2个frm的示例，方便内部使用。
- 你可以提供这些函数：
  - Tag(byval Name as string) as cToast : 用于在集合对象中给每个示例命名。如果用户没调用这个函数，那么show 的时候就随机生成一个8位字符的名称（你自己写个函数）。
  - CloseMe(byval Name as string) as bool : 用于关闭指定命名的窗体，如果成功，就从集合中删除该实例。
  - CloseAll() as bool : 关闭当前实例中的所有窗体。
