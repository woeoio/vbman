# 软件著作权申请填写说明

## 一、申请流程概览

```
1. 准备材料 → 2. 网上填报 → 3. 提交材料 → 4. 受理审查 → 5. 获得证书
```

## 二、所需材料清单

### 2.1 必需材料

| 序号 | 材料名称 | 份数 | 说明 |
|------|----------|------|------|
| 1 | 软件著作权登记申请表 | 1份 | 在线填写后打印 |
| 2 | 软件源程序代码 | 1份 | 前后各30页，共60页 |
| 3 | 软件文档 | 1份 | 使用说明书或设计文档 |
| 4 | 身份证明文件 | 1份 | 身份证复印件或营业执照 |

### 2.2 身份证明材料要求

**个人申请：**
- 身份证复印件（正反面）
- 需本人签字

**公司申请：**
- 营业执照副本复印件（加盖公章）
- 如委托办理，需授权书

## 三、申请表填写要点

### 3.1 软件基本信息

| 字段 | 填写建议 |
|------|----------|
| 软件全称 | 建议：VBMAN网络应用开发框架软件 |
| 软件简称 | VBMAN |
| 版本号 | V1.0（首次申请建议用V1.0）|
| 开发完成日期 | 填写实际完成日期 |
| 首次发表日期 | 可选择"未发表"保留权利 |

### 3.2 软件用途与技术特点

**用途**：简要描述软件的主要用途，100字以内  
**技术特点**：列举3-5个技术特点，50字以内

可参考本文档 `01_软件基本信息.md` 中的内容。

## 四、源代码准备规范

### 4.1 基本要求
- 提交源程序前30页和后30页，共60页
- 每页不少于50行代码
- 不足60页的需全部提交

### 4.2 格式要求
- 使用A4纸张大小
- 页眉标注软件名称和版本号
- 页码连续编制
- 代码清晰可读

### 4.3 注意事项
- 不要注释掉代码来凑行数
- 不要包含空行（连续空行只算一行）
- 第30页和第31页要连续
- 最后一页要是程序结束符

## 五、文档准备规范

### 5.1 可选文档类型
1. **使用说明书**：适合应用程序类软件
2. **设计说明书**：适合系统软件、工具软件

### 5.2 文档要求
- 提交前30页和后30页，共60页
- 每页不少于30行文字
- 不足60页的需全部提交
- 包含软件功能说明、操作流程、界面截图等

## 六、申请方式

### 6.1 网上申请
1. 登录中国版权保护中心官网：https://www.ccopyright.com.cn/
2. 注册账号并实名认证
3. 在线填写申请表
4. 上传电子版材料
5. 邮寄纸质材料

### 6.2 代理申请
- 可委托专业代理机构办理
- 费用约300-800元不等
- 可节省时间和精力

## 七、费用说明

| 项目 | 费用 |
|------|------|
| 登记费 | 免费（2023年起） |
| 证书费 | 免费（电子证书） |
| 加急费 | 视加急程度收取 |
| 代理费 | 300-800元（如委托代理） |

## 八、审查周期

| 申请类型 | 审查周期 |
|----------|----------|
| 普通申请 | 约30-60个工作日 |
| 加急申请 | 约10-20个工作日 |

## 九、注意事项

1. **软件名称**：建议包含品牌名+功能描述+软件/系统字样
2. **版本号**：首次申请建议用V1.0
3. **发表状态**：选择"未发表"可保留更多权利
4. **源代码**：确保前后30页是连续的代码
5. **文档**：要与源代码对应的软件版本一致

## 十、常见问题

**Q1: VB6开发的软件能申请软著吗？**  
A: 可以，任何编程语言开发的软件都可以申请。

**Q2: 组件库/框架能申请软著吗？**  
A: 可以，只要具有独立功能即可。

**Q3: 使用了开源代码会影响申请吗？**  
A: 只要在申请表中如实说明，并遵守开源协议，一般不会影响。

**Q4: 申请被驳回的常见原因？**  
A: 1) 材料不齐全；2) 源代码与文档不符；3) 软件描述不清晰。
